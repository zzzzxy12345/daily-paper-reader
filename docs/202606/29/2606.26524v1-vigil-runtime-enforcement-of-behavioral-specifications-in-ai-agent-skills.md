## 摘要
代理系统日益通过第三方技能运作，允许模型生成的决策影响文件、通信渠道和网络物理设备。这些技能通常包含自然语言规范，定义访问权限、披露限制、执行权限和所需前提条件。尽管此类规范描述了技能行为的预期边界，但它们本身并未提供可执行的运行时执行。执行这些规范带来了上下文粒度挑战：即使策略是为特定任务上下文编写的，监控器仍必须决定观察哪些事件、保留什么状态、推理到执行的哪个阶段以及在何处干预。选择错误的粒度可能要么阻止良性执行，要么遗漏仅在多个动作中出现的违规行为。然而，大多数现有执行机制假设固定的事件模型或执行点。

在这项工作中，我们提出了VIGIL，一个用于代理系统的端到端运行时执行框架。VIGIL根据技能规范中的行为策略、操作员定义的约束以及跨多个技能的全局规则来检查代理的实际执行轨迹。为了使这些策略可执行，VIGIL引入了一种策略语言，该语言捕获了代理-工具事件上的特定上下文执行要求，包括时间依赖关系、参数约束和值流条件。该语言与符号评估规则配对，这些规则将策略转化为有限轨迹上的SMT约束，使VIGIL能够检测依赖于事件顺序、参数关系或跨调用值流的违规行为，而不是依赖固定的单调用过滤器。在涵盖办公文档、操作和工程任务的实际LLM代理运行中，VIGIL以超过95%的召回率和低于10%的误报率检测到策略违规。

## Abstract
Agentic systems increasingly act through third-party skills, allowing model-generated decisions to affect files, communication channels, and cyber-physical devices. These skills often include natural-language specifications that define access permissions, disclosure limits, execution privileges, and required preconditions. Although such specifications describe the intended boundaries of skill behavior, they do not by themselves provide executable runtime enforcement. Enforcing them raises a contextual granularity challenge: even when a policy is written for a particular task context, a monitor must still decide which events to observe, what state to retain, how far across the execution to reason, and where to intervene. Choosing the wrong granularity can either block benign executions or miss violations that emerge only across multiple actions. Most existing enforcement mechanisms, however, assume a fixed event model or enforcement point.   In this work, we present VIGIL, an end-to-end runtime enforcement framework for agentic systems. VIGIL checks an agent's actual execution trace against behavioral policies from skill specifications, operator-defined constraints, and global rules spanning multiple skills. To make such policies executable, VIGIL introduces a policy language that captures context-specific enforcement requirements over agent-tool events, including temporal dependencies, argument constraints, and value-flow conditions. The language is paired with symbolic evaluation rules that translate policies into SMT constraints over finite traces, allowing VIGIL to detect violations that depend on event order, argument relationships, or cross-call value flow rather than relying on fixed single-call filters. On real LLM-agent runs spanning office-document, operational, and engineering tasks, VIGIL detects policy violations with over 95% recall and a false-positive rate below 10%.

- **研究动机**：大语言模型（LLM）驱动的代理系统正通过第三方“技能”（skills）执行敏感操作（如读写文件、操作仓库、控制物理设备）。这些技能通常附有自然语言编写的行为规范（如权限、前置条件、禁止操作），但**运行时缺乏自动执行**——代理自身不可信，且现有防御机制存在结构性盲区。
- **核心问题**：违反规范的攻击往往隐藏在**多个工具调用之间**（例如：跳过验证步骤后直接消费未验证的中间产物），单个调用的语法检查或权限门控无法发现这种跨动作的时序依赖违规。现有方法要么只能检查单一动作（如AgentSpec、Progent），要么离线分析（如LLM裁判），无法在运行时做出确定性阻断。
- **整体意义**：VIGIL首次提出一个**端到端运行时参考监控器**，将自然语言行为规范编译成可执行策略，基于有限执行轨迹进行SMT求解，在违规发生前阻止并定位证据。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：严格分离规范的通用含义与每次运行的具体实例化。通过**轨迹抽象**（将原始工具调用转化为有类型的事件序列）和**策略接地**（将抽象规范映射到具体动作、参数、值），将时序安全属性转化为**量词自由的SMT公式**，一次求解即可判定整条轨迹是否满足策略，当不满足时返回最小不可满足核（unsat core）作为违规证据。
- **关键技术细节**：
  - **策略语言**（图4）：支持六种时序形式——`abs`（禁止）、`prec`（前置条件）、`resp`（义务）、`bresp`（有界响应）、`rslv`（最终状态）、`until`（区间守卫），以及通用`□Θ`。模式（pattern）可匹配动作名、参数值、输出、状态，并允许变量绑定以跨事件关联相同值。
  - **符号编译**（图5）：每条策略语句`φ`被编译为对每个事件`e`的条件`Rφ(e)`，最终合取形成SMT查询。编译规则保证所有量词都限定在有限事件集上，公式是量词自由的。
  - **SMT检查**：主过程见Algorithm 1。先`ABSTRACT`提取签名Σ和事实基`JτK`，再`COMPILE`将规范D编译为策略P（使用LLM辅助但仅限词汇表内），然后逐条检查：若`unsat(JτK ∧ Rφ)`则记录违规事件（从muc中提取）。若无一不满足则返回`safe`。
  - **局部化**：求解器返回的最小不可满足核直接指出是哪个事件上的哪个条件导致了冲突，从而精确定位违规调用。
  - **实际部署**：在待定调用前执行检查，若不安全则阻止该调用，并返回违规证据供回滚/重规划/人工审查。

## 6. 主要结论与发现
- **主结果**：在SB+SI上，VIGIL达到**95.8%召回率、89.6%精度、92.6% F1**，领先最强基线LLM-as-judge达**15.8个F1点**（AgentSpec 57.6%、Progent 50.3%、LLM-as-judge 76.8%）。

- **平衡误差**：VIGIL的宏观BEE仅5.4%，远低于AgentSpec（15.2%）和LLM-as-judge（12.5%），证明其在减少漏报和误报上均显著改进。
- **真实世界发现**：在216条技能包执行中确认**34条策略违规**，包括NVIDIA Slurm无限重提、Trail of Bits ASan缺少内存限制、Databricks CLI版本门缺失等，NVIDIA已承认缺陷。

- **运行时开销**：端到端平均4.38s，其中92.5%来自一次LLM编译调用（可缓存优化），实际SMT检查仅0.27s，对现代agent工作流（分钟级）可忽略。

## 7. 优点
- **方法创新**：首次将**有限轨迹上的时序安全属性**、**变量绑定**和**SMT求解**结合用于agent运行时监控，实现了跨动作的上下文感知。
- **端到端自动化**：从自然语言规范自动编译为可执行策略，无需手工编写监控代码；编译与验证分离，LLM仅在受限词汇表内参与。
- **可解释性**：违反时返回具体违规事件（通过unsat core），便于回顾和纠正。
- **实际有效性**：不仅检测到基线方法遗漏的违规，还在生产级技能生态中发现并确认了真实缺陷，证明了实际需求。
- **全面实验**：覆盖多种基准、消融、真实世界部署、不同错误类型分析，验证了方法的鲁棒性和泛化能力。

## 8. 不足与局限
- **假阳性来源**：策略编译的精度依赖于LLM翻译质量。实验中约8个假阳性中有4个来自编译器忽略了自然语言中的修饰词（如“批量”操作为“编辑”），导致无冲突更新被误报。
- **假阴性来源**：当策略相关行为发生在**内部脚本**（如Python或shell）中且未暴露为独立工具调用时，VIGIL无法观测。如`secure_archive.py`在脚本体内删除文件而未产生trace事件，导致违规未被检测。这是系统边界的固有限制。
- **编译成本**：端到端4.38s中LLM调用占92.5%，虽然可以缓存策略、使用更快的推理后端或小型模型，但当前原型依赖外部闭源模型（claude-opus-4-6），可能带来延迟和成本问题。
- **未覆盖的无限轨迹**：策略基于有限轨迹检查，对于可能无限延续的实时系统（如持续感知循环）需要额外设计结束条件或增量检查。
- **威胁模型假设**：假设trace收集器是可信且完整的，但若攻击者能够操纵收集器（如通过OS级干扰），则可能绕过监控。
- **实验局限**：SB+SI的规模相对较小（152条），真实世界研究虽然产品级但仅获取了216条执行，更广泛的市场技能还需进一步测试。另外，未进行跨不同LLM编译器（如GPT-4、本地模型）的敏感性分析。
