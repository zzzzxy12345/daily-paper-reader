## 摘要
混合代理（MoA）架构通过将多个LLM代理组织成分层推理流水线来改善推理时扩展。然而，现有的MoA变体随着深度增加无法维持收益，表现出退化、早期平台期或饱和。我们提出ReM-MoA，一种内存增强的MoA框架，通过两种机制维持扩展：（1）排名推理记忆，持久存储并利用比较评审代理对所有层的推理轨迹进行排名；（2）策展多样化内存路由方案，将不同代理暴露于成功和失败轨迹的不同组合，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域评审蒸馏流水线，通过前沿模型监督提高排名质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准上，ReM-MoA在深度和宽度扩展上均持续优于先前的MoA变体，且其优势随深度增加而扩大，将结构化跨层推理记忆确立为可扩展多代理推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有的 Mixture-of-Agents (MoA) 架构通过将多个 LLM 代理组织成分层推理流水线来改善推理时计算扩展，但它们在增加深度时普遍出现性能退化、早期平台期或饱和等“扩展崩溃”现象。具体表现为三种故障模式：原始拼接传播错误、局部层间通信丢失早期高质量推理、层内代理因上下文趋同而丧失探索多样性。
- **动机**：实现可扩展的多代理推理需要同时保持**质量**（高质量推理轨迹的持久化与传播）和**多样性**（层内代理不陷入冗余路径）。现有方法（如标准 MoA、RMoA、AttentionMoA）均未同时满足这两个需求。
- **整体含义**：论文提出**结构化跨层推理记忆**是维持 MoA 扩展的关键缺失机制，并通过 ReM-MoA 框架首次实现了随深度增加性能持续提升。

## 2. 方法论：核心思想与关键技术细节

### 核心思想
ReM-MoA 将 MoA 扩展视为一个**质量-多样性联合保持**问题，而非单纯的聚合或交互问题。通过两个核心组件实现：
1. **排序推理记忆（Ranked Reasoning Memory, M）**：持久存储所有层的推理轨迹，并利用一个比较性评审代理（Reviewer Agent）对每层的 N 条轨迹进行评分（1-10 整分数）和附带针对性理由（rationale），形成结构化记忆条目。
2. **策展多样化内存路由（Curated Diversified Memory Routing, Π）**：从第 2 层开始，每个提议代理（proposer agent）从记忆中获得一个**定制的参考集**，该参考集由三种路由函数之一生成：Top n（高分）、Bot n（低分）、Con n（对比组合：Top a ∪ Bot b）。不同代理获得不同的参考集，从而维持层内探索多样性。

### 关键技术细节
- **记忆子系统**：每层提交一个条目 M_l = {(r_{l,j}, s_{l,j}, ρ_{l,j})}_{j=1}^N，记忆是追加式的，支持后续任意层的跨层读取。
- **评审代理**：离线使用 Qwen2.5-7B-Instruct（默认），可选通过**前沿模型蒸馏**（GPT-5.5 作为教师）进行 LoRA 微调，得到 ReM-MoA* 版本。蒸馏训练数据来自 ReM-MoA 自身运行产生的（问题、轨迹）对，教师给出评分和理由作为监督目标。
- **路由方案实例**（N=3）：一个代理使用 Con 2 (Top1∪Bot1)，一个使用 Top 2，一个使用 Bot 2。更大 N 时按比例扩展（附录表 5）。
- **聚合器**：最后层级后，聚合器也通过三种路由从记忆中获得参考集，综合生成最终答案。

### 公式/算法流程（文字说明）
1. 输入问题 x。
2. 对于第 1 层：每个提议代理直接基于 x 生成推理轨迹 r_{1,j}；评审代理对 1 层所有轨迹评分并给出理由；记忆 M_1 存储 (r, s, ρ)。
3. 对于第 2 层至第 L 层：每个提议代理 A_{l,j} 根据路由 Π_j 从 M_{≤l-1} 中获取参考集 S_{l,j}（包含轨迹、评分、理由），与 x 拼接后生成推理轨迹 r_{l,j}；评审代理再次评分；记忆追加 M_l。
4. 最后，聚合器从最终记忆 M_{≤L} 中通过三种路由获取参考集，合成最终答案。

## 6. 主要结论与发现

1. **ReM-MoA 持续随深度和宽度提升**：在所有 5 个基准上，ReM-MoA 和 ReM-MoA* 均优于全部 baseline，且优势随深度增加而扩大（图 3 显示单调扇出效应）。在深度 L=9 时，ReM-MoA 在形式逻辑上比 AttentionMoA 高 7.3 个百分点，在 CRUX 上高 6.3，在 MATH 上高 4.1。
2. **标准 MoA 退化，RMoA 平台期，AttentionMoA 饱和**：实验结果复现了三种故障模式，验证了之前分析。
3. **跨层访问和路由多样化是关键**：消融实验表明，限制为仅相邻层访问（类似 RMoA）导致最大性能下降（平均 -4.28 点），共享参考集（去多样化）是第二大贡献因素（平均 -2.50 点），而去除理由影响较小（-1.52 点）但保留趋势。
4. **评审蒸馏提升了性能且能跨域迁移**：Multi-FT 在所有基准上进一步提升，LOO（留出一个域训练）恢复了 Multi-FT 平均增益的 44%，证明蒸馏学到了可转移的评估技能。
5. **结构化跨层推理记忆是可持续扩展的关键缺失机制**。

## 7. 优点

1. **问题定义清晰，动机有力**：论文首先通过实验可视化三种故障模式，建立质量-多样性联合保持的需求，引出设计需求 Q 和 D，逻辑链完整。

## 8. 不足与局限

1. **宽度探索有限**：宽度仅到 N=8（与 prior work 一致），未验证更大宽度下的行为。理论上 ReM-MoA 在宽度上应也能持续扩展，但缺乏证据。
2. **提议代理尺度单一**：仅使用 7-8B 模型，未测试是否适用于更大（30B-70B）或更小（1-3B）模型。质量-多样性权衡可能随代理能力变化而改变，泛化性待验证。
3. **评审代理开销与偏见**：每层一次评审调用（共 L 次），虽比 AttentionMoA 低，但仍增加总调用数。评审代理本身是 LLM，可能继承位置偏差、自偏好等偏见。论文通过比较评分和对比路由减轻但未消除这些偏见。
4. **计算成本缺乏完整披露**：未提供全量实验的总 GPU 小时数，仅给出蒸馏训练时长。对于重现或部署，读者需自行估算。
5. **高价值应用的安全风险**：论文承认评审和提议可能产生事实错误或社会偏见，高 stakes 部署需额外防护。这也是所有 LLM 驱动的框架的共同局限。
6. **蒸馏依赖前沿模型访问**：ReM-MoA* 需要访问商业模型（GPT-5.5）作为教师，这可能带来访问成本或隐私问题。但作者强调蒸馏是可选的，无蒸馏的 ReM-MoA 已超越所有基线。
