## 摘要
深思熟虑需要时间。在实时环境中，这段时间并非免费。标准强化学习（RL）回避了这一点，因为环境会无限期等待智能体的决策。相反，我们研究的是实时RL环境，其中环境在等待智能体动作时会继续推进。基于先前的实时形式化，我们引入了变延迟实时强化学习，其中智能体在每个决策点自行决定思考时间长度，因为环境仍在推进。对于我们所使用的规划智能体，合适的延迟是状态相关的，而天真地规划如何规划可能会使智能体瘫痪。我们通过训练一个轻量级的门控策略来覆盖规划器，以选择状态依赖的规划预算。在实时吃豆人、俄罗斯方块、贪吃蛇、Speed Hex和Speed Go中，我们的门控策略优于固定预算和启发式基线，并能迁移到环境和智能体运行在两个不同GPU上的实时设置中。

## Abstract
Deliberating takes time. In real-time settings, that time is not free. Standard reinforcement learning (RL) sidesteps this as the environment waits indefinitely for the agent's decision. Instead, we study real-time RL environments where the environment progresses while waiting for the agent's action. Building on prior real-time formalizations, we introduce variable-delay real-time RL, where the agent chooses how long to deliberate at each decision point since the environment progresses. For the planning agents we use, the right delay is state-dependent, and naively planning how long to plan can paralyze the agent. We instead approach this setting by training a lightweight gating policy on top of a planner to select state-dependent planning budgets. Across real-time Pac-Man, Tetris, Snake, Speed Hex, and Speed Go, our gating policy outperforms fixed-budget and heuristic baselines, and transfers to a real-time setup where the environment and agent run on two different GPUs.

- **研究动机**：标准强化学习（RL）假设决策环境会无限等待智能体输出动作，但在实时环境中，智能体思考时环境会持续推进（如幽灵移动、重力下落、对手计时）。因此，思考时长本身是成本：一个完美的动作如果迟了一步可能不如一个及时的好动作。
- **核心问题**：智能体需要在每个决策点自适应地决定“花多少时间思考”，即选择规划预算。但“思考如何思考”可能陷入元推理悖论——规划本身也需要时间。
- **整体含义**：本文首次将实时RL中的延迟选择形式化为一个学习问题，挑战了传统RL中“计算免费”的假设，揭示了在受限资源下理性分配规划预算的重要性。

## 2. 方法论：核心技术细节

### 核心思想
- 提出**变延迟实时强化学习**框架：智能体是一个“运行在时间中的程序”，每帧必须输出一个动作；思考的时间以环境进展（帧数或时钟消耗）的形式支付。
- 将问题建模为**半马尔可夫决策过程（SMDP）上的预算选项**：每个选项 `ok` 表示用 `k` 帧时间进行规划，其中前 `k-1` 帧执行快速反射策略（πreflex）的承诺动作，第 `k` 帧应用MCTS规划的改进动作。
- 训练一个**轻量级门控策略**（gating policy）在冻结的AlphaZero规划器之上，根据状态选择规划预算 `k`。

### 关键技术细节
- **反射策略**：使用规划器的原始策略网络（无MCTS）的单次前向传播（约2ms），实时输出动作。
- **预算选项结构**：MCTS在模拟中显式rollout承诺动作，使得搜索动作作用于未来状态，与真实执行一致。
- **门控网络架构**：输入包括游戏观察、规划器的空间特征（卷积+残差块）和标量值估计；输出离散分布 over `k`。
- **训练算法**：使用PPO，并修改GAE以处理变长折扣（每元步折扣 `γ^k`，其中 `k` 为实际环境帧数）。
- **效率优化**：使用JAX实现，通过 `vmap`、`jit`、`lax.scan` 等将MCTS内循环和PPO rollout全部编译为单一XLA内核，避免Python开销。

## 6. 主要结论与发现

1. **学习规划预算优于固定预算**：门控策略在全部5个环境中超越所有固定预算和启发式基线。例如Pac-Man中比最佳固定预算（k=3）提升10.3%，实时Tetris提升65%。
2. **门控策略学习到状态依赖的分配**：在危险/紧追状态下（如鬼怪靠近、棋盘填满、蛇身受限）选择短预算快速反应；在安全/可预见状态下（如鬼怪远离、空棋盘）选择长预算深入规划。时钟环境下，小时间预算时偏向快速选项，大时间预算时更均匀分配。
3. **仿真训练可直接迁移到异步硬件部署**：使用承诺动作协议的仿真训练策略在真实双GPU异步执行中保持性能，延迟在H100和A100上基本在预算内。
4. **缩放关系验证**：更多MCTS模拟带来更高动作质量，但也增加延迟，因此存在明确的权衡（图3）。
5. **冻结规划器选择重要**：基模型的选择显著影响最终性能（附录F），轻量级门控策略无法补偿低质量规划器。

## 7. 优点

- **框架创新**：将变延迟实时RL形式化为SMDP上的预算选项，比固定延迟框架（Ramstedt & Pal, 2019）更一般化，且避免状态增广带来的维度灾难。
- **方法实用**：门控策略极轻量（单次前向），避免“规划如何规划”的元推理悖论，且训练简单（只优化门控，规划器冻结）。
- **实验全面**：覆盖五种不同实时机制的环境，跨评估和消融设计严谨，部署验证贴近实际应用。
- **实用贡献**：开源完整代码和预训练checkpoint，附带可复现脚本（附录A）。
- **迁移成功**：仿真训练的协议直接用于硬件部署，无需任何调整，证明承诺动作模拟是准确的抽象。

## 8. 不足与局限

- **依赖完美模拟器**：承诺动作协议假设MCTS可以rollout真实未来，需环境模型精确（AlphaZero型）；若扩展到学习动力学模型（如MuZero），rollout精度下降，协议效果可能打折。
- **冻结规划器限制**：门控策略无法补偿基模型的质量（附录F），且未能联合优化规划器和门控，可能不是整体最优。
- **预算集手调**：预算集 `K` 是手工选择的离散值（每个环境4个），未探索连续预算或自适应预算终止（如值函数阈值）。
- **环境范围有限**：实验仅在离散动作的游戏环境中验证，未涉及连续控制（如机器人、自动驾驶）或部分观测环境。
- **门控网络输入设计**：仅使用了规划器的一层空间特征，更丰富的内部状态（如搜索树信息）可能进一步改进。
- **超时机制简化**：主实验中时钟耗尽后使用πreflex作为回退，降低了难度；严格超时控制显示问题更简单（图12），说明任务设计对结论有影响。
- **缺乏理论分析**：未给出门控策略的收敛性保证或最坏情况偏差边界。
