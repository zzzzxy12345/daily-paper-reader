## 摘要
智能体导航系统需要一个基础导航模型，其观察策略可以在推理时从外部重新配置，因为指令跟随、目标搜索、目标跟踪和自动驾驶共享相同的感知规划主干，但需要根本不同的策略来处理视觉流。我们提出了Qwen-RobotNav，这是一个基于Qwen-RobotNav构建的可扩展导航模型，通过一个具有两个互补维度的参数化接口来解决这一问题：多个选择导航行为的任务模式，以及控制视觉历史如何编码的可控观察参数（例如，token预算、每个摄像头的权重）。通过对所有参数进行训练时随机化，Qwen-RobotNav对任何推理时配置都具有鲁棒性，且无需对Qwen-RobotNav主干进行任何架构修改。我们在1560万个样本上训练Qwen-RobotNav；与视觉语言数据共同训练防止了仅在轨迹训练中观察到的崩溃为反应式动作序列映射器。参数化接口还使Qwen-RobotNav成为智能体系统的自然构建模块：对于长期场景，上层规划器将目标分解为子任务，并在情节中动态切换Qwen-RobotNav的任务模式和上下文策略，通过重复调用同一模型来组合复杂行为。大量实验表明，Qwen-RobotNav在主要导航基准测试中取得了新的最佳结果。该模型在从2B到8B参数规模上表现出良好的扩展性，联合多任务训练发展出跨任务家族迁移的共享空间规划基础，并在多种环境中展示了针对真实世界机器人的强零样本泛化能力。

## Abstract
Agentic navigation systems require a base navigation model whose observation strategy can be externally reconfigured at inference time, because instruction following, object search, target tracking, and autonomous driving share the same perception-planning backbone yet demand fundamentally different strategies for consuming the visual stream. We present Qwen-RobotNav, a scalable navigation model built on Qwen-RobotNav that addresses it through a parameterised interface with two complementary dimensions: multiple task modes that select the navigation behaviour, and controllable observation parameters (e.g., token budget, per-camera weights) that govern how visual history is encoded. With training-time randomization over all parameters, Qwen-RobotNav is robust to any inference-time configuration requiring zero architectural modification to the Qwen-RobotNav backbone. We train Qwen-RobotNav on 15.6M samples; co-training with vision-language data prevents the collapse into reactive action-sequence mappers observed in trajectory-only training. The parameterised interface also makes Qwen-RobotNav a natural building block for agentic systems: for long-horizon scenarios, an upper-level planner decomposes goals into sub-tasks and dynamically switches Qwen-RobotNav's task mode and context strategy mid-episode, composing complex behaviours from repeated calls to the same model. Extensive experiments show that Qwen-RobotNav sets new state-of-the-art results across major navigation benchmarks. The model exhibits favourable scaling from 2B to 8B parameters, with joint multi-task training developing a shared spatial-planning substrate that transfers across task families, and demonstrates strong zero-shot generalisation to real-world robots across diverse environments.

# Qwen-RobotNav 技术报告：面向智能体导航系统的可扩展导航模型

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现代机器人导航面临多样化的任务（指令跟随、目标搜索、目标跟踪、自动驾驶），这些任务共享感知-规划主干，但对视觉观测流的消费策略有根本不同的需求（例如，指令跟随需要长时间记忆，而目标跟踪只需最近几帧）。现有统一导航模型（如NavFoM、ABot-N0）采用固定的上下文策略（均匀子采样或滑动窗口），无法在推理时根据任务动态调整，也无法集成到上层智能体系统中。
- **整体含义**：为了构建可交互的智能体导航系统，需要一个**可重配置**的基础导航模型——其观测策略能够被外部规划器在推理时动态选择，而无需任务特定的重训练。Qwen-RobotNav 通过参数化接口实现了这一目标，使上层规划器能够根据子目标切换任务模式和观测配置，从而在长期复杂任务中组合行为。

## 2. 提出的方法论

### 核心思想
- **将多任务导航挑战重新定义为观测上下文建模问题**：通过暴露一个参数化接口，使外部规划器可以动态控制模型如何消费视觉历史，而不是针对不同任务设计不同的模型架构。
- **训练时随机化所有观测参数**：确保模型对推理时的任意配置都具有鲁棒性，无需任务特定的调优。
- **语言级标识**：相机身份、时间顺序、具身类型完全通过自然语言标签传达到基座大模型中，无需架构修改。

### 关键技术细节
1. **模型架构**：基于 Qwen3-VL（图像编码器 SigLIP-2 + LLM 主干），增加轻量级4层MLP动作头，输出8个航点（每个3自由度：x,y,θ）。所有视觉-语言共同训练，保持开放世界的感知能力。
2. **任务自适应观测编码**：四个控制轴：
   - **视觉 token 预算 B**：总视觉 token 数量（2048–4096）。
   - **时间衰减 γ**：控制对近期帧的偏置（1–3）。
   - **相机权重 w_c**：每个相机的重要性乘数（如前置权重更高）。
   - **帧采样模式 m**：随机采样或最新帧采样。
   
   具体算法：先子采样 T' 帧→计算时间权重 ω_t = exp(γ·t/(T'-1))→联合权重矩阵 W[t,c]=ω_t·w_c→约束分配：每帧最小 b_min 个 token，剩余预算按比例分配，超过上限 b_max 的释放并重新分配。
3. **视角与时间标识**：在视觉 token 前插入自然语言标签，如“Time step t Camera: Front <image>”，使 LLM 自动关联空间语义。
4. **具身感知提示设计**：通过系统提示中的自然语言前缀（如“Imagine you are a robot…”）区分不同平台（机器人、汽车等）。
5. **行动规划**：轻量4层MLP将LLM最终隐藏状态映射为24维输出（8航点×3）。
6. **训练策略**：
   - 复合损失：L = L_traj + λ·L_VL（L_traj为MSE损失，L_VL为下一个 token 预测损失）。
   - 所有观测参数每样本独立随机化。
   - 训练数据：85%导航轨迹规划数据 + 15%导航相关视觉语言推理数据。
   - 优化器 AdamW（β1=0.9, β2=0.95, weight decay 1e-2），余弦学习率，峰值2e-5（视觉和LLM）和1e-4（动作头）。

### 智能体系统集成
- **上层规划器**（Qwen3.6-Plus）将长期任务分解为子目标，动态选择任务模式（VLN、PointNav、ObjNav、Tracking）和观测配置，调用 Qwen-RobotNav 执行。
- **轨迹证据压缩**：每次执行生成紧凑证据记录（子目标、进度、关键帧索引等），维护跨情节记忆笔记，支持上下文压缩。

## 7. 优点

1. **参数化接口设计创新**：将观测上下文控制作为第一类可配置自由度，允许多任务共享模型权重，上层规划器可动态切换配置。
2. **自然语言标识**：利用预训练LLM的语义理解，避免额外学习，保持架构简洁且可扩展（只需新提示模板）。
3. **大规模数据生成**：包括自动T2V数据生成管线，减少对模拟器的依赖，增强视觉多样性。
4. **智能体系统集成完善**：提供轨迹证据压缩、跨情节记忆、视觉工具调用接口，形成完整的上层-下层分工架构。
5. **实验验证全面**：覆盖仿真和真实世界，多个任务族和规模，消融充分。

## 8. 不足与局限

1. **计算资源依赖**：8B模型需要2.8K H100小时，算力门槛较高；边缘部署仍需Jetson Thor，且延迟略高于云端（5.1Hz vs 4.9Hz）。
2. **跟踪任务成功率较低**：在EVT-Bench上虽然跟踪率最高（90.0%），但成功率（77.4%）低于ABot-N0（86.9%）和TrackVLA++（86.0%），表明多任务训练可能引入保守行为。
3. **自动驾驶AlpaSim表现弱**：零样本下离专业模型（Alpamayo-R1）差距大，表明对特定驾驶场景的泛化仍需改进。
4. **Token分配算法经验性**：作者承认当前分配算法是启发式的，未使用更原则性的方法（如可学习注意力分配）。
5. **实验偏差**：OVON评估中Qwen-RobotNav使用单目相机，而ABot-N0使用全景输入，比较上存在不公平。尽管如此Qwen-RobotNav仍超越部分指标，但结论需谨慎。
6. **长期记忆依赖文本摘要**：证据笔记主要通过文本存储，可能丢失关键视觉细节，虽可通过关键帧ID回溯，但增加了规划器负担。
7. **环境多样性**：训练数据主要来自Matterport3D/HM3D等室内场景和nuScenes/OpenScene等城市驾驶，极端环境（如室外越野、水下）未覆盖。
