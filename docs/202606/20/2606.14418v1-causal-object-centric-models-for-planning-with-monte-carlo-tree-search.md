## 摘要
我们提出了COMET（用于高效树搜索的因果物体中心模型），一种基于模型的强化学习算法，在槽结构潜空间中执行蒙特卡洛树搜索。COMET将冻结的无监督物体中心编码器与基于Transformer的世界模型配对，其中通过一种新颖的动作-槽融合机制将动作绑定到物体上，该机制用于槽转移预测。策略和价值头使用物体因果注意力，通过学习的每槽相关性分数调节令牌交互，使决策集中在任务相关实体上。COMET为MuZero风格的潜空间规划添加了显式的物体级归纳偏置。在来自Object-Centric Visual RL基准测试、ManiSkill、Robosuite和VizDoom的八个视觉和动态多样任务中，与物体中心和整体基线相比，COMET在训练早期阶段实现了更高的平均归一化分数。

## Abstract
We introduce COMET (Causal Object-centric Model for Efficient Tree search), a model-based reinforcement learning algorithm that performs Monte Carlo Tree Search in a slot-structured latent space. COMET pairs a frozen unsupervised object-centric encoder with a transformer-based world model, in which actions are bound to objects through a novel action-slot fusion mechanism that is used in slot transition prediction. Policy and value heads use object-causal attention, modulating token interactions by learned per-slot relevance scores so that decision-making concentrates on task-relevant entities. COMET adds an explicit object-level inductive bias to MuZero-style latent planning. Across eight visually and dynamically diverse tasks from the Object-Centric Visual RL benchmark, ManiSkill, Robosuite, and VizDoom, COMET achieves a higher mean normalized score during the early stages of training compared to object-centric and monolithic baselines.

### 2. 方法论
- **整体框架**：COMET 遵循 MuZero 风格的潜空间规划范式，但用一组物体槽（slots）替代单一潜状态。状态表示为 `\bar{s}_t = {s^1_t, ..., s^n_t}`，其中 `n` 为预设的最大物体数+背景槽。
- **关键组件**：
  - **物体中心编码器 (Slots Extractor)**：使用预训练的物体中心模型（如 SLATE、DINOSAUR、Slot Contrast）从观测中提取槽，并在训练过程中冻结。为保证时序一致性，初始化下一时间步的槽为当前时间步的槽。
  - **Transformer 世界模型**：基于 nanoGPT 架构，在每个时间步内，将槽表示 `\bar{s}_t` 和槽条件动作嵌入 `\bar{a}_t` 作为输入序列。通过特定的注意力掩码（slots 之间可以相互注意，action 嵌入注意所有 slots）进行建模，预测下一时刻的槽 `\hat{s}_{t+1}` 和奖励 `\hat{r}_t`。动作与槽的融合通过“动作-槽绑定”机制实现：将动作嵌入与每个槽拼接后经共享 MLP 投影，得到槽条件动作嵌入。
  - **策略和价值网络**：使用物体因果注意力（Object-Causal Attention），其中引入一个可学习的目标令牌，以及每个槽的因果分数 `α^i_t` 表示该槽对当前决策的相关性。通过一个基于因果图（目标、因果物体、非因果物体）和权重矩阵 `W_t` 的注意力调制机制，使策略和价值预测聚焦于重要物体。
  - **损失函数**：联合优化下一状态预测（MSE）、奖励预测（交叉熵）、策略预测（交叉熵）和价值预测（交叉熵），如公式(5)所示。

### 6. 主要结论与发现
- COMET 在训练早期（约前 200k-300k 步）的平均归一化分数高于所有基线，显示出更好的样本效率。
- 在视觉简单任务（如 Object Goal, Object Comparison）上，COMET 利用高质量物体表示和因果注意力显著优于其他方法；在复杂动态任务（如 Defend the Line）中达到竞争性能；在视觉和控制都困难的任务（如 Block Lifting）中表现中等，可能与编码器无法完美分割物体有关。
- 总体表明：物体级归纳偏置有助于提升 MuZero 风格规划的效率和聚焦能力。

### 7. 优点
- **新颖的动作-槽绑定机制**：将动作与每个槽独立融合，避免了单一动作嵌入瓶颈，使世界模型能逐槽预测下一状态。
- **物体因果注意力**：通过学习因果分数动态聚焦于任务相关物体，提高了策略和价值的可解释性与有效性。
- **统一 Transformer 架构**：将状态、动作、预测统一在一个框架内，符合 MuZero 风格，易于扩展。
- **跨任务验证**：在 8 个视觉和动态多样任务上验证了方法，性能稳定，平均归一化分数高。

### 8. 不足与局限
- **物体中心编码器依赖**：当前无监督编码器在复杂场景（如 Block Lifting 中物体与背景融合）中分割质量不足，限制了 COMET 的性能上限。
- **计算开销**：Transformer 的自注意力复杂度为 `O(n^2)`（`n` 为槽数），在处理大量物体时扩展性差。
- **因果注意力机制的适用范围**：在需要间接因果推理的任务（如 Object Interaction 中通过推物体间接达成目标）中，因果注意力对直接相关物体敏感的假设可能不适用。
- **缺乏消融研究**：未对核心组件（如动作绑定、因果注意力、编码器选择）进行系统消融，使得贡献归因不够清晰。
- **训练数据规模限制**：编码器训练仅使用随机策略收集的数据（30万-50万帧），在更复杂环境中可能不足以泛化。
