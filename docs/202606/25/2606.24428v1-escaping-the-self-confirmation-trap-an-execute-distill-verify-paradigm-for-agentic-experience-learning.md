## 摘要
经验驱动的自我进化对于大型语言模型（LLM）智能体通过开放世界交互进行改进至关重要。然而，现有的经验学习方法大多依赖单智能体循环，其中同一智能体执行任务、总结结果并确定记忆内容。这种设置使智能体容易陷入“自我确认陷阱”：错误但自洽的轨迹被误认为是成功经验，导致在检索和复用过程中产生累积性错误。为解决这一问题，我们提出了EDV，一种用于可靠经验学习的执行-提炼-验证框架。在执行阶段，多个异构智能体并行探索同一任务空间，生成多样化的候选轨迹。在提炼阶段，一个专门的第三方智能体比较分析这些轨迹，生成候选经验，减少以执行者为中心的总结偏差。在验证阶段，执行组通过共识机制验证候选经验，只有通过验证的经验才被写入共享或私有记忆。通过解耦这三个阶段，EDV将经验学习从孤立的自我反思转变为协作构建，在记忆插入前过滤错误和噪声内容。我们在三个具有挑战性的长周期基准测试上评估了EDV：tau2-bench、Mind2Web和MMTB。结果表明，EDV持续优于强基线，验证了可靠的经验构建对于稳健的智能体自我进化至关重要。我们的代码可在https://github.com/shidingz/EDV获取。

## Abstract
Experience-driven self-evolution is critical for large language model (LLM) agents to improve through open-world interaction. However, existing experience learning methods mostly rely on single-agent loops, where the same agent executes tasks, summarizes outcomes, and determines memory content. This setup makes agents vulnerable to the Self-Confirmation Trap: wrong-but-self-consistent trajectories are misidentified as successful experience, leading to cumulative errors during retrieval and reuse. To address this issue, we propose EDV, an Execute-Distill-Verify framework for reliable experience learning. In the Execute stage, multiple heterogeneous agents explore the same task space in parallel to generate diverse candidate trajectories. In the Distill stage, a dedicated third-party agent comparatively analyzes these trajectories to produce candidate experiences, reducing executor-centric summarization bias. In the Verify stage, the execution group validates candidates via a consensus mechanism, and only approved experiences are written into shared or private memory. By decoupling the three stages, EDV transforms experience learning from isolated self-reflection into collaborative construction, filtering erroneous and noisy content before memory insertion. We evaluate EDV on three challenging long-horizon benchmarks: tau2-bench, Mind2Web and MMTB. Results show EDV consistently outperforms strong baselines, validating that reliable experience construction is essential for robust agent self-evolution. Our code is available at https://github.com/shidingz/EDV.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的大语言模型（LLM）智能体经验学习方法大多采用单智能体闭环——同一智能体执行任务、总结结果、决定写入记忆。这种设计在缺乏显式真实反馈的开放世界环境中极易陷入“自我确认陷阱”（Self-Confirmation Trap）：智能体会将错误但自洽的轨迹误判为成功经验，并写入记忆，后续检索和复用导致错误累积放大。
- **整体含义**：经验学习的根本挑战不仅是收集更多经验，而是如何更可靠地构建经验，在记忆插入前抑制错误和噪声内容。作者指出，鲁棒的智能体自我进化不仅依赖于更丰富的记忆，更依赖于记忆构建过程本身的可靠性。

## 2. 方法论：EDV框架

- **核心思想**：通过解耦执行、提炼和验证三个角色，将经验学习从单智能体的自我反思循环转变为多智能体协作的经验构建与过滤过程，从而打破自我确认陷阱。
- **三个阶段**：
  - **Execute（执行）**：从异构智能体池中随机采样一个执行组（通常2个智能体），每个智能体独立与同一任务环境交互，生成多样化的候选轨迹。异构性（不同基座模型或提示策略）有助于扩大解空间覆盖，暴露不同的成功/失败模式。
  - **Distill（提炼）**：指定一个第三方蒸馏智能体（从池中随机选取，与执行组不同），对多个轨迹进行对比分析，识别跨轨迹的有用差异，提炼出可复用的候选经验（1~3条）。蒸馏智能体不做简单的轨迹摘要，而是提取可泛化的规则，减少执行者自总结带来的偏差。
  - **Verify（验证）**：执行组的原始智能体作为验证者，对每条候选经验基于自身执行上下文进行二元判断（通过/不通过）。采用严格默认拒绝策略：
    - 获得全体一致通过的经验 → 写入**共享记忆库**（Shared Memory Bank）
    - 获得部分通过的经验 → 写入相应智能体的**私有记忆库**（Private Memory Bank）
    - 其余丢弃。
- **存储与推理使用**：
  - 维护**能力矩阵**（Ability Matrix），记录各智能体在不同任务上的表现，用于推理时任务-模型匹配。
  - 推理时，新任务先通过能力矩阵选择最合适求解器，然后进行层次检索：先查共享库，若不足再查该求解器的私有库，将检索到的记忆附加到任务上下文中。

## 6. 主要结论与发现

1. **EDV 持续优于所有基线**：在 τ2-bench 上 Pass@1 达到 86.6（Router 83.5，Judge 81.5，NM 最差 76.4）；在 Mind2Web 和 MMTB 上也有类似优势。
2. **写出的记忆质量更高**：人类审计显示，EDV 在正确性、可操作性、特异性上显著高于 RB，噪声/幻觉和复用危害得分更低。
3. **自我确认陷阱真实存在**：注入10%错误记忆使 RB 在 Retail 域 Pass@1 从82.5降至77.2，证实错误记忆的危害。
4. **解耦设计的关键性**：渐进消融表明单智能体自验证无用甚至有害；多智能体+第三方蒸馏比执行者自蒸馏更有效；共识验证提供最终质量保障。
5. **能力矩阵和记忆层次均有贡献**：移除能力矩阵下降2.0个点，移除共享/私有记忆层次下降2.7~2.9个点；私有记忆覆盖约三分之一任务，互补共享记忆。
6. **EDV 训练更稳定**：随 epoch 增加持续提升，而 RB 在 epoch 2 后下降。
