## ABSTRACT 

Conversational AI agents in Customer Experience (CX) typically suffer from a Closed-World Constraint , ignoring high-velocity external shifts like viral trends or outages. Ad-hoc web search attempts to bridge this gap but often introduce prohibitive latency and context poisoning . We introduce PulseCX , a framework that decouples knowledge acquisition from consumption. Adopting a structure-first paradigm, PulseCX employs an asynchronous agent to linearize signals into a Decay-Aware Temporal Knowledge Graph (DA-TKG) governed by reinforcement–decay dynamics to actively manage information lifecycles. By coupling this self-evolving memory with hierarchical intent gating, PulseCX removes synchronous search bottlenecks ( <10ms overhead) and drives significant gains in Intent Resolution (IRR) and Customer Satisfaction (s-CSAT) in dynamic environments. 

## 1 INTRODUCTION 

Large Language Models (LLMs) have transformed automated Customer Experience (CX), enabling agents that reason and act across complex workflows. Yet most deployed agentic systems suffer from a fundamental disconnect: they are grounded in static internal artifacts (manuals, FAQs, dashboards) while operating in a world shaped by fast-moving external events. This gap is most visible during high-impact moments—such as influencer-driven PR crises or early-stage safety incidents (e.g., widespread reports of device overheat-ing)—where customer perception shifts long before internal systems update. We define this mismatch as the Closed-World Constraint : the misalignment between an agent’s static internal state and high-velocity external reality. In practice, this manifests as context blindness in Dynamic intents , where successful resolution depends on external, time-sensitive knowledge. Common attempts to bridge this gap via web search or news APIs frequently introduce unacceptable latency and severe context poisoning , flooding the inference window with noisy, dangerous or contradictory signals which may become security risks Chen et al. (2024a); Eslami & Yu (2025). To address this challenge, we propose PulseCX , a framework for selectively injecting real-time CXM context into agentic systems without sacrificing reliability. PulseCX adopts a structure-first paradigm (Xu et al., 2025), utilizing an asynchronous discovery loop to linearize raw external signals into structured Context Objects before they enter the reasoning window. Building on recent work on agent memory Hu et al. (2025), we store these objects in a Decay-Aware Temporal Knowledge Graph (DA-TKG) governed by reinforcement–decay dynamics. This allows the system to actively maintain the lifecycle of facts—amplifying persistent signals (e.g., outages or fraud campaigns) while allowing transient noise to fade. Together, these mechanisms enable agents to resolve Dynamic intents while preserving low latency, bounded noise and poisoning signals , and reduced hallucination risk. 1

## 2 RELATED WORK 

Agent Memory. Prior work on memory for LLM-based agents emphasizes short-term, episodic, or long-term recall for coherence and planning, typically assuming a static external world Wang et al. (2024); Hu et al. (2025). Related efforts augment LLMs with temporal knowledge by injecting facts from predefined Temporal Knowledge Graphs into prompts or aligning model representations with TKG embeddings (e.g., TimeR4 Chen et al. (2024b) and GenTKGQA Gao et al. (2024b)), but these approaches primarily target temporal reasoning for QA tasks rather than maintaining a lightweight, continuously evolving world-state memory for real-time agent grounding. PulseCX instead targets environment factual memory , explicitly modeling the lifecycle of external events via reinforcement–decay dynamics. Retrieval-Augmented and Online Agents. Retrieval-Augmented Generation (RAG) and tool-using agents extend LLMs with external knowledge at inference time Lewis et al. (2020); Yao et al. (2023). While effective for factual lookup, these approaches often incur latency and introduce noise in dynamic settings. PulseCX decouples context acquisition from inference by asynchronously structuring external signals into bounded, low-latency context. Related work in other ML systems have similarly explored gated mechanisms to control how contextual signals influence downstream optimization (Gao et al., 2024a). Context in CX Systems. Most deployed CX systems rely on session history, customer profiles, and business rules. Although effective for routine interactions, they struggle with 

Dynamic intents driven by rapidly evolving external events. While prior work has emphasized the importance of real-time context in CX systems Kim (2022); Motevalli & Razavi (2024); Vashishth et al. (2024); Badheka (2014), such capabilities remain largely absent in practice. PulseCX addresses this gap by selectively injecting real-time situational context only when required. Overall, PulseCX formalizes real-time external context as a first-class, evolving memory substrate rather than an ad-hoc retrieval artifact. 

## 3 METHODOLOGY : T HE PULSE CX F RAMEWORK 

PulseCX decouples high-latency knowledge acquisition from real-time inference via a two-stage architecture: 

an Offline Asynchronous Stage that structures world knowledge into a Decay-Aware Temporal Knowledge Graph (DA-TKG) , and an Online Synchronous Stage that selectively grounds user queries using this dynamic substrate, preserving contextual awareness of the "wild" without the latency or noise of synchronous web search. 23.1 STAGE 1: O FFLINE WORLD MODELING (S TRUCTURE -F IRST )To capture high-velocity signals without slowing down inference, we employ a periodic Social Search Agent (SSA) that scans external sources at fixed intervals (e.g., T = 15 m). This stage transforms raw noise into structured intelligence through two phases. 

Phase 1: Linearization and Context Object Creation. Raw streams from web search or social APIs are unstructured and noisy. PulseCX enforces a Structure-First paradigm: the SSA output is immediately linearized into a structured Context Object . This object encapsulates not just the information, but its provenance and scope. By formalizing the input as a tuple O = {Type , Scope , Confidence , State }, we ensure that only high-quality signals enter the graph (see Appendix A for details). 

Phase 2: Memory Evolution (Reinforcement-Decay). Unlike static databases, the DA-TKG is self-evolving, utilizing a Reinforcement-Decay mechanism to manage node lifecycles. To prune stale context (e.g., expired sales), node weight W decays over time t:

Wt = Wlast · e−λ(t−tlast ) and Wnew = min(1 .0, W decayed + β · Cconf ) (1) where volatility coefficient λ is high for viral trends and low for infrastructure persistence. Conversely, reinforcement constant β ensures that signals re-confirmed by the SSA (with confidence Cconf ) gain salience while one-off noise naturally fades. This ensures the graph accurately reflects the current reality rather than historical noise. 3.2 STAGE 2: O NLINE SYNCHRONOUS GROUNDING 

When a user queries the system, PulseCX executes a high-speed retrieval pipeline designed to inject relevant context without introducing latency or hallucinations. The process begins with a Hierarchical Gating Engine 

that classifies the user’s intent. Static intents (e.g., “Reset Password”) bypass the graph entirely to prevent context poisoning, while Dynamic intents (e.g., “Why is the site slow?”) trigger retrieval. To select the most pertinent information, we employ a Salience-Weighted Semantic Reranker that scores candidate nodes ( N )by combining their semantic similarity to the query ( Q) with their current decay-adjusted weight ( Wt): 

R(N ) = Sim (Q, N ) · Wt (2) This scoring function ensures the agent prioritizes nodes that are both relevant and temporally active, naturally filtering out expired events like old flash sales. The top-ranked nodes are then synthesized into a deterministic 

Context Card —a structured briefing injected into the main agent’s system prompt (see Appendix B). Finally, the system closes the loop via a Query Refiner that analyzes chat logs for information gaps (e.g., unknown slang), generating targeted search queries for the Social Search Agent’s next offline cycle. 

## 4 EXPERIMENTAL EVALUATION 

We evaluate PulseCX as a specialized grounding layer designed to bridge the “Closed-World” gap. Our primary hypothesis is that by decoupling intelligence gathering (Offline) from context injection (Online), PulseCX can resolve dynamic user intents without the latency penalties or hallucination risks inherent to naive web-search agents. 4.1 EXPERIMENTAL SETUP AND BASELINES

PulseCX is evaluated against two baselines: (1) Static RAG , the industry standard utilizing fixed manuals, and (2) a Naive Online Agent ,which executes synchronous web searches 1 per query. ## 5 CONCLUSION 

PulseCX bridges the gap between static internal knowledge and dynamic reality by formalizing unstructured signals into a Decay-Aware Temporal Knowledge Graph (DA-TKG). Our results demonstrate that environment factual memory is essential for navigating the volatility of the real world. However, some limitations remain: (1) Temporal blindness bounded by the fixed cycle time ( T ), which may obscure events occurring between updates; (2) vulnerability to adversarial poisoning via coordinated inauthentic activity; and (3) reliance on heuristic decay parameters ( λ) rather than learned coefficients. In conclusion, PulseCX demonstrates that the Closed-World Constraint of modern agents is an architectural choice, not a fundamental limitation. The Context Object is the fundamental unit of the DA-TKG, created offline by the Social Search Agent (SSA). It translates unstructured social signals into a provenance-aware format that preserves uncertainty and scope. A.1 SCHEMA DEFINITION 

Each object O is stored as a structured metadata tuple to allow for rapid mathematical weighting and filtering without LLM intervention. 

ContextObject = {type: {crisis, brand, influencer, trend}, entity_scope: {brand, product, region, industry}, confidence: numeric (0.0 - 1.0), summary_state: "Natural language description of the event", source_ref: "Traceable pointer to the original signal", initial_weight: numeric (assigned based on source volume) }

A.2 GRAPH INJECTION 

When the SSA identifies a signal (e.g., a competitor flash sale), it populates this schema. This object is then injected into the DA-TKG as a node. The Memory Evolution logic (Phase 2) then manages this object’s weight over time, ensuring that the "State" is only considered for retrieval as long as it remains salient.
