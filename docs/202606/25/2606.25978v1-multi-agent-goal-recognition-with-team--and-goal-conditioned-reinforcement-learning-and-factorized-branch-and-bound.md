## 摘要
多智能体目标识别要求观察者联合推断哪些智能体共同行动以及每个团队试图实现什么目标，因此假设空间随团队划分和每个团队的目标数量组合增长。真实应用如无人机监控和协作机器人只暴露智能体的轨迹，迫使观察者仅从行为中对团队-目标假设进行排序。基于分支定界的多智能体目标识别（MAGR-BB）通过一个共享的团队与目标条件策略作为评分模型，在分解的分支定界搜索中解决此问题。在受控的多智能体Blocksworld基准测试中，MAGR-BB在整个轨迹中返回与穷举搜索相同的排名最高的假设，同时将假设具体化数量减少数个数量级，并显著降低累积识别运行时间。

## Abstract
Multi-agent goal recognition asks an observer to jointly infer which agents act together and what each team is trying to achieve, so the hypothesis space grows combinatorially with the number of team partitions and goals per team. Real applications such as drone surveillance and collaborative robotics expose only the agents' trajectory, which forces the observer to rank team-goal hypotheses from behavior alone. Multi-Agent Goal Recognition with Branch-and-Bound (MAGR-BB) addresses this setting with a shared team- and goal-conditioned policy used as the scoring model inside a factorized branch-and-bound search. On a controlled multi-agent Blocksworld benchmark, MAGR-BB returns the same top-ranked hypothesis as exhaustive search throughout the trajectory while cutting hypothesis materialization by orders of magnitude and reducing cumulative recognition runtime substantially.

- **现有方法不足**：多数目标识别针对单智能体（如规划类识别器、GRAQL等）；多智能体方法依赖显式计划库、动作模型或BDI意图结构，缺乏基于学习行为策略的同时推断团队与目标的方法。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **MAGR-BB**：一种无模型（model-free）识别器，结合两个关键组件：①一个共享的**团队与目标条件策略网络**（Team- and Goal-Conditioned Policy）作为评分模型；②一个**分解的分支定界搜索**（Factorized Branch-and-Bound），利用可加性得分在保持最优性的前提下大幅剪枝。

### 关键技术细节

#### (1) 形式化定义
- 智能体集合A = {1,...,N}，隐藏的团队划分P = {T₁,...,T_K}（带索引标签），每个团队槽k有候选目标集G_k。完整假设h = (P, g₁,...,g_K)。
- 观察者获得部分轨迹O_{1:t} = {(s_τ, a_τ)}，其中a_τ = (a¹_τ,...,a^N_τ)。
- 假设观察者能完全观测联合状态和动作（部分可观测留作未来工作）。

#### (2) 团队与目标条件策略
- 构建一个**反事实观测**（counterfactual observation）o_i^t(T, g)，由四个固定维度特征块拼接：领域状态特征、智能体自身标识、团队成员掩码、目标编码。保持世界快照不变，仅替换候选团队和目标。
- 使用**Transformer编码器**（2层、4头、隐藏维度256）策略π_θ(a_i^t | o_i^t(T,g), c_i^t)，其中c_i^t是团队内自回归的队友动作上下文（同一联合步中已选动作）。
- 训练方式：使用PPO（CTDE框架，动作裁剪0.2，学习率1e-5，批大小256，每轮4个epoch，γ=0.99），采用基于长度和难度的分阶段课程训练（6个阶段）。
- 对比GRAQL：GRAQL为每个候选目标学习独立策略；本文共享一个网络处理所有(T,g)对。

#### (3) 反事实评分
- 对于假设h，计算累计得分S_t(h) = Σ_{τ=1}^{t} Σ_{i=1}^N log max(π_θ(a_i^τ | o_i^τ(h), c_i^τ(h)), ε_score)，其中ε_score=1e-10防止log(0)。
- **非竞争性得分条件**（Definition 1）：每个团队的目标得分仅依赖其自身的团队和目标，不依赖其他团队的候选目标。因此完整假设得分可分解为各团队局部得分之和：S_t(P,g₁,...,g_K) = Σ_{k=1}^K S_t(T_k, g_k)。
- 轨迹终止时添加一致性惩罚（λ_term=2.0）。

#### (4) 分解分支定界识别（Algorithm 1）
- **核心思想**：维护局部团队-目标得分表，避免实时构建笛卡尔积。采用三级剪枝：
  - **评分阶段剪枝**（Scoring-stage prune）：若分区的陈旧上界低于当前top-k堆的最小值，跳过刷新。
  - **分区级剪枝**（Partition-level prune）：刷新后计算分区上界U_t(P)=Σ_k max_g S_t(T_k, g)，若仍低于阈值，跳过该分区。
  - **局部级剪枝**（Local-level prune）：在分区内，将每个团队的目标按局部得分降序排序，使用优先队列以最佳优先顺序生成完整假设；当堆顶得分≤阈值时停止扩展。
- **Proposition 1**：在非竞争性得分条件下，上述剪枝不会丢弃得分高于当前top-k阈值的假设，因此与穷举搜索返回相同排名。

## 3. 实验设计：数据集/场景、基准、对比方法

### 场景与数据集
- **领域**：多智能体Blocksworld（经典规划领域，7块积木，初始和目标配置为堆栈）。
- **环境配置**：两个隐藏团队，每队2个智能体，各在独立工作空间（7块积木）操作。使用分离工作空间以控制满足非竞争性得分条件，并非算法前提。
- **候选目标**：每个团队槽的候选目标为长度为2-4的有序堆栈，共1092个（42+210+840=1092）。
- **候选划分**：4个智能体分成2对，共6个带索引标签的划分。

### 对比方法
- **Factorized Exhaustive**（无剪枝的穷举基准）
- **Scoring Prune**（仅启用评分阶段剪枝）
- **Partition B&B**（仅启用分区级剪枝）
- **Local B&B**（仅启用局部级剪枝）

## 8. 不足与局限

### 实验覆盖
- **单一领域**：仅Blocksworld，且为分离工作空间（非必要条件但简化），是否适用于共享资源、连续动作空间、更高维状态空间未知。
- **小规模智能体**：仅2个团队×2个智能体，未扩展到更大规模（如4个团队、8个或更多智能体）。组合爆炸更严重时分支定界的剪枝效果可能下降。
- **完全可观测假设**：观察者获得完整联合状态-动作，部分可观测（如只获得部分智能体轨迹或噪声观测）未考虑。
- **不足的对比基准**：未与基于计划库的多智能体方法（如MAPRAP、Algorithm X等）进行定量对比，仅与自身的穷举版本对比。
- **统计样本小**：每个配置仅5条轨迹，未给出置信区间或显著性检验，可能不足以得出普适结论。

### 潜在偏差风险
- **训练与识别使用相同域名**：策略在同一个Blocksworld环境中训练，在相同环境上识别，可能存在分布内过拟合。对未见过的目标分布或不同初始状态分布的泛化能力未测试。
- **噪声生成方式**：动作噪声为随机均匀采样，而非更真实的“错误动作”模型（如执行相似但错误的操作），可能过于理想化。

### 应用限制
- **离线训练前提**：需要预先学习和训练策略，对于新领域或动态变化的环境重新训练成本高。
- **非竞争性得分条件**：虽然非强制要求，但在共享工作空间或资源竞争场景下，团队间得分可能相互影响，分解性质不成立，需要修改评分或剪枝策略。
- **分支定界产生的假设数仍可能随团队数指数增长**：在更大规模时，即使剪枝也可能需访问较多分区，实时性仍面临挑战。
