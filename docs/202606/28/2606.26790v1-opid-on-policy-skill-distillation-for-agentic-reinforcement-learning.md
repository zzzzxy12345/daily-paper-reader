## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励对于哪些中间决策应被强化或抑制提供的指导甚少。在策略自蒸馏提供了密集的词元级监督，然而现有的技能条件变体通常依赖于外部技能记忆或检索到的特权上下文，这些维护成本高昂，且在多轮交互中可能与当前策略产生的状态分布不匹配。我们提出了OPID（On-Policy Skill Distillation），一个直接从已完成在策略轨迹中提取技能监督的框架。OPID将轨迹事后知识表示为分层技能：回合级技能捕捉全局工作流或失败规避规则，而步骤级技能在关键时间步捕捉局部决策知识。一种关键优先路由机制在识别到关键决策时使用步骤级技能，否则默认使用回合级技能作为引导。将所选技能注入交互历史，使旧策略能够在原始上下文和技能增强上下文下对同一采样响应重新评分。由此产生的对数概率偏移提供了词元级自蒸馏优势，该优势与结果优势结合用于策略优化。因此，OPID保留了强化学习作为主要训练目标，同时引入了密集、分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的问答上的实验表明，OPID在智能体性能、样本效率和鲁棒性上普遍优于仅结果强化学习和现有技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题定义**：基于结果的强化学习（如 GRPO）提供了稳定的训练目标，但其奖励稀疏且延迟，仅给出轨迹级成功/失败信号，无法辨别哪些中间决策导致了最终结果。这在长程多步交互中尤为严重。
- **现有局限**：在策略自蒸馏方法虽能提供密集 token 级监督，但依赖于外部技能记忆库或检索到的特权上下文，维护成本高，且检索的技能可能与当前策略的在线状态分布不匹配（状态漂移）。
- **本文目标**：提出 **OPID**（On-Policy Skill Distillation），无需外部技能库，直接从当前策略完成的在线轨迹中提取事后（hindsight）技能，并蒸馏回策略参数，使强化学习同时获得轨迹级奖励和密集、分布匹配的 token 级引导。

## 2. 方法论
### 2.1 核心思想
- 将每一条完成的在线轨迹转化为**分层技能**：
  - **回合级技能**：总结全局工作流（成功轨迹）或失败规避规则（失败轨迹）。
  - **步骤级技能**：在关键时间步提供的具体局部决策知识，如“避免重复无效操作”、“下一步应检查的物品”。
- 通过**关键优先路由**机制，为每个决策步选择最合适的技能粒度：关键步骤用步骤级技能，其余用回合级技能。
- 将路由后的技能注入交互历史，让旧策略在原始上下文和技能增强上下文下对同一响应重新评分，利用对数概率差计算**技能自蒸馏优势**。
- 技能优势与 GRPO 风格的结果优势相结合，用于策略优化（PPO clipped loss）。

### 2.2 关键技术细节
- **技能提取**：使用 LLM analyzer 处理轨迹记录（任务描述、观测、响应、状态反馈、终端结果），输出结构化 JSON：`episode_skill` 及 `step_skills`（对应关键步骤索引）。
- **关键优先路由**：若当前步 `t` 被 analyzer 标识为关键步，则使用步骤级技能 `s_step`；否则使用回合级技能 `s_ep`。
- **技能注入**：将技能拼接到原始上下文 `h` 中形成增强上下文 `˜h`，原响应 `y` 不变。
- **技能优势**：`A_skill = log π_θ_old(y | ˜h) - log π_θ_old(y | h)`，正值表示 token 与技能一致。
- **最终优势**：`A_OPID = A_ep + λ_skill * A_skill`，其中 `A_ep` 为轨迹在组内归一化后的结果奖励优势。
- **策略损失**：标准 PPO clipped 目标，附加 KL 正则项。

### 2.3 算法流程（文字说明）
1. 采样一组 N 条 on-policy 轨迹。
2. 对每条轨迹计算回合级优势（基于组内结果奖励均值/标准差）。
3. 通过 analyzer 提取回合级技能和关键步骤级技能。
4. 遍历轨迹每一步：
   - 判断是否关键步，选择对应技能。
   - 注入技能得到增强上下文。
   - 使用旧策略计算原上下文和增强上下文下的 token 对数概率，求得 token 级技能优势。
   - 将回合级优势广播到所有 token，并与技能优势线性组合。
5. 对所有 token 的最终优势应用 PPO clip 更新策略。

## 6. 主要结论与发现
1. **OPID 一致优于纯结果 RL**：相较于 GRPO，在 3B 骨干上 ALFWorld +9.3，Search QA +8.6，WebShop +10.9；7B 骨干上分别为 +8.8、+7.2、+7.1。

3. **技能内化，无需推理时技能输入**：OPID 在无技能 prompt 下评估仍显著优于 Skill-GRPO*（有技能 prompt），说明技能被吸收到参数中。

5. **跨域泛化好**：在 unseen 环境中 OPID 超越了 GRPO +7.7 点。
6. **层级技能和关键优先路由均不可或缺**：消融证实两者贡献互补，路由优于简单叠加。

## 8. 不足与局限
- **实验覆盖有限**：未测试更大更强模型（如 70B+ 或闭源 API），也未在更复杂环境（WebArena、VisualWebArena、OdysseyArena、AgentBench）上评估。仅覆盖具身、购物、搜索三种场景。

- **超参数敏感性**：λ_skill、max critical steps 等超参数可能需针对具体场景调整，论文未提供详细鲁棒性分析。
- **技能可能过拟合特定轨迹**：从行为策略轨迹提取的技能可能强化当前策略的行为偏差，减少探索多样性，长期可能陷入次优。
