## 摘要
对于多步交互环境中的LLM智能体而言，一个关键挑战是如何有效利用积累的交互经验。现有工作通常将这种经验的两种用途分开：将其以自然语言规则的形式保留在模型之外，供后续提示使用；或者利用轨迹和反馈来更新模型参数。前者易于解释，但可能与不断发展的策略不同步；后者能更广泛地改进策略，但在稀疏奖励设置中，对局部错误的纠正能力有限。我们提出了面向LLM智能体的经验规则与策略的联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则库和策略。在决策时，JERP检索与任务相关的规则，并将这些规则与交互历史一起作为智能体的条件。每个回合结束后，它利用收集到的轨迹，既优化策略，又通过将当前推出的结果与参考成功轨迹进行比较来修订规则库。这种耦合使规则库与不断发展的策略保持一致，同时允许将稳定有效的行为逐渐吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了一致的提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

- **研究背景**：大型语言模型（LLM）在多步交互环境（如网页导航、具身交互、工具使用）中面临如何有效利用积累的交互经验的挑战。
- **现有方法的局限性**：

- **本文核心问题**：能否利用**同一批交互数据同时支持显式规则更新和策略优化**，使规则与策略持续对齐，互补优缺点。
- **整体含义**：提出JERP（Joint Learning of Experiential Rules and Policies），将经验规则库和策略参数在同一训练循环中联合学习，实现规则可动态适应策略，策略可吸收稳定有效的行为。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 为每个任务实例维护一个**长期经验规则池** `K(d)`，包含规则文本 `z_i` 和效用分数 `s_i`。
- 每回合开始前，从规则池中按效用分数降序选择 top-k 条规则作为**工作规则集** `ĨK(d)`，与任务描述和交互历史拼接作为 LLM 的输入。
- 智能体与环境交互生成一个轨迹组 `T_d = {τ_i}`（共 N 条轨迹）。
- 同一轨迹组被用于**两阶段优化**：
  - **阶段 I：模型参数更新**——基于组相对优势计算 GRPO 损失，更新策略参数（使用 LoRA）。
  - **阶段 II：经验规则更新**——通过**对比反射**（对比当前轨迹组与参考成功轨迹）生成规则编辑操作（ADD/EDIT/UPVOTE/DOWNVOTE/MERGE），更新规则文本和效用分数，并修剪低分规则。

### 关键技术细节
- **轨迹采样**：工作规则集在单个回合内固定，避免时步级规则切换导致上下文漂移。规则按分数排序后填入固定模板。
- **模型参数更新（阶段 I）**：
  - 沿用 GRPO 的组相对优势估计：对轨迹组内奖励标准化得到优势 `A_i`，同一轨迹所有时步共享优势。
  - 损失函数为 GRPO 目标（含 clip 和 KL 正则），更新 LoRA 参数。
- **经验规则更新（阶段 II）**：
  - 输入：当前规则池 `K(d)`、当前轨迹组 `T_d`、参考成功轨迹集 `T^+(d)`。
  - 利用一个冻结参数的 LLM 作为反射模块，生成结构化编辑操作列表（五种操作）。
  - 根据操作执行规则文本修改和分数调整（新规则初始分 s₀>0，UPVOTE/DOWNVOTE 加减固定增量，MERGE 取最大值）。
  - 分数低于阈值的规则被移除，规则池保持容量限制。
- **算法流程（Algorithm 1）**：对每个任务初始化规则池和成功轨迹集；每轮采样任务实例 → 构建工作规则集 → 采样 N 条轨迹 → 计算优势 → 更新策略参数 → 反射生成编辑操作 → 应用规则更新 → 更新成功轨迹集。

## 3. 实验设计：数据集/场景、Benchmark、对比方法

### 环境和评估指标
- **AlfWorld**：基于文本的家庭操作环境，包含6种任务类型——Pick（捡放）、Clean（清洁后放置）、Heat（加热后放置）、Cool（冷却后放置）、Look（观察物体）、Pick2（捡两个物体）。评估指标：任务成功率（%）。
- **WebShop**：在线购物交互环境，用户根据需求搜索、浏览、比较并购买商品。评估指标：平均分数（Average Score）和任务成功率（%）。

### 对比方法
| 方法 | 类型 | 说明 |
|------|------|------|
| Vanilla LLM | 直接提示 | 无推理、无记忆、无参数更新 |
| ReAct | 推理+行动交替 | 当前回合内推理，无跨回合经验 |
| Reflexion | 反思记忆 | 将失败反思写入外部记忆，供后续回合使用 |
| RLOO | 策略优化（REINFORCE） | 使用留一基线估计优势，更新参数 |
| GRPO | 策略优化（组相对） | 组内奖励标准化估计优势，更新参数（JERP的参数量部分类似） |
| JERP（本文） | 联合学习 | GRPO参数更新 + 经验规则池动态更新（均使用LoRA） |

### 实现细节
- 所有强化学习方法均使用 **LoRA** 进行参数高效微调（秩64，缩放因子64）。
- 每组实验独立运行3次取平均。
- 超参数：学习率3e-6，轨迹组大小 N=8（AlfWorld）/ 8（WebShop），折扣因子0.95，训练轮数200（AlfWorld）/300（WebShop），批次大小256（AlfWorld）/64（WebShop），每回合最大步数50（AlfWorld）/15（WebShop）。

1. **JERP 在两种环境上均取得最佳性能**：

4. **联合学习使规则池与策略保持动态对齐**：策略更新后，规则池通过对比反射吸收新的有效行为模式，同时废弃或修正过时规则。

## 7. 优点

- **方法创新**：巧妙地结合了外部规则维护和参数优化两种范式，在同一训练循环中相互促进，避免各自缺点。
- **规则可解释性**：规则以自然语言存储，可被人类审查和调整，增加透明度。
- **自动化规则维护**：通过对比反射机制自动产生添加、编辑、合并、评分调整等操作，无需人工干预。
- **鲁棒性**：在多个复杂交互任务上一致优于强基线（GRPO本身已是领先方法），证明方法有效。
- **消融实验干净**：直接验证了核心机制的必要性。
- **使用 LoRA 高效微调**，节省算力。

## 8. 不足与局限

- **规则检索简单**：当前仅按效用分数全局排序取 top-k，未考虑任务实例的细粒度相关性（如不同子任务可能需要不同规则）。论文承认“这是实现选择，并非一定优于更细粒度的实例级检索”。
- **规则更新依赖额外 LLM 调用**：需要冻结参数的 LLM 生成编辑操作，增加推理开销和延迟，且可能不稳定（LLM 输出的质量影响规则池）。
- **实验场景有限**：仅在文本仿真环境（AlfWorld、WebShop）评估，未涉及多模态、真实机器人、长对话等更复杂场景，通用性待验证。
- **超参数敏感性未分析**：如规则池容量、初始分数 s₀、增减分增量 δ±、合并阈值等对性能的影响未知，可能导致在不同任务上需要调参。
- **计算效率未比较**：虽然性能提升，但 JERP 多出的规则更新步骤的计算开销未被量化，与 GRPO 的比较未考虑时间成本。
