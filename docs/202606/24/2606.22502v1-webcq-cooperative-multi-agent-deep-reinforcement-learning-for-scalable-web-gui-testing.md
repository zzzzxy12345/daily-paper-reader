## 摘要
基于多智能体强化学习（MARL）的技术在GUI测试中显示出前景。然而，随着现代GUI软件复杂性的增加，现有的基于MARL的方法（如MARG和Fastbot）由于其底层表格强化学习算法的固有限制而难以扩展。这限制了它们对大规模商业GUI软件的适用性，尤其是具有巨大状态空间和众多交互元素的Web应用。为了填补这一空白，我们提出了WebCQ，一种新颖的基于MARL的可扩展Web GUI测试方法。WebCQ集成了QTRAN用于多智能体协调以及轻量级同步机制，使其能够在异步Web测试场景下工作。它为每个UI事件提取语义和探索特征以形成动作向量。该向量与当前状态向量拼接并输入策略网络，从而在动态动作空间中实现基于DQN的决策。我们在八个大型商业网站上评估了WebCQ。在相同时间预算和智能体数量下，WebCQ比MARG探索了33.3%更多的状态，执行了42.2%更多的不重复动作，同时在八个被测网站中有六个触发了更多故障。它还展示了强大的可扩展性，在20小时的实验中保持更高的动作吞吐量，并且随着智能体数量的增加实现了更大的性能提升。这些结果表明，WebCQ克服了现有基于MARL方法的关键限制，为增强现代Web GUI测试提供了可扩展且有效的解决方案。

## Abstract
Multi-agent reinforcement learning (MARL)-based techniques have shown promise for GUI testing. However, as the complexity of modern GUI software increases, existing MARL-based approaches (e.g., MARG and Fastbot) struggle to scale due to the inherent limitations of their underlying tabular reinforcement learning algorithms. This limits their applicability to large-scale commercial GUI software, especially web applications with vast state spaces and many interactive elements. To fill this gap, we propose WebCQ, a novel MARL-based approach for scalable web GUI testing. WebCQ incorporates QTRAN for multi-agent coordination and a lightweight synchronization mechanism, allowing it to work under asynchronous web testing scenarios. It extracts semantic and exploration features for each UI event to form an action vector. This vector is concatenated with the current state vector and fed into the policy network, enabling DQN-based decision making within a dynamic action space. We evaluated WebCQ on eight large-scale commercial websites. Under the same time budget and agent count, WebCQ explored 33.3% more states and executed 42.2% more unique actions than MARG, while triggering more failures on six of the eight websites under test. It also demonstrated strong scalability, maintaining higher action throughput during 20-hour experiments, and achieving greater performance improvements as the number of agents increased. These results show that WebCQovercomes key limitations of existing MARL-based approaches, providing a scalable and effective solution for enhancing modern web GUI testing.

### 论文结构化总结：WebCQ

#### 1. 论文的核心问题与整体含义（研究动机与背景）

*   **核心问题**：现有基于多智能体强化学习（MARL）的Web GUI测试方法（如MARG、Fastbot）在面对现代大规模、高复杂度的Web应用时，存在严重的**可扩展性**问题。
*   **根本原因**：这些方法底层依赖**表格型强化学习算法**（如Q-learning）。表格方法需要为每个状态-动作对维护一个Q值表，当Web应用的状态空间和交互元素数量剧增时，Q表会迅速膨胀，导致**状态爆炸**和巨大的**沟通开销**，从而限制了其应用。
*   **研究空白**：虽然已有研究尝试将深度强化学习（DRL）用于GUI测试，但它们多是**单智能体**范式，性能受限于孤立学习的局限性。而多智能体方法又受困于表格算法的瓶颈。因此，需要一种既能利用DRL处理高维状态的能力，又能通过MARL实现高效协作的新方法。

#### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：提出 **WebCQ**，一个基于**合作式多智能体深度强化学习**的新框架，旨在解决大规模Web GUI测试的可扩展性问题。它采用**集中式训练与分布式执行（CTDE）** 范式，并引入先进的MARL算法 **QTRAN** 来实现智能体间更有效的协调。

*   **关键技术细节**：
    *   **问题形式化**：将多智能体Web GUI测试任务建模为**去中心化部分可观测马尔可夫决策过程（Dec-POMDP）**，其中每个智能体只能观察到自己的网页状态（局部观察），但共同优化一个全局探索目标。
    *   **DQN驱动的分布式执行**：
        *   **状态向量**：提出一种新颖的**标签-深度** (tag-depth) 的HTML DOM树表示方法。通过计算每个HTML标签在DOM树中的平均深度，生成固定维度的状态向量，既高效又适合神经网络输入。
        *   **动作向量**：为每个UI事件构建一个丰富的动作向量，包含三类特征：**文本语义**（如按钮文字与“同意”、“提交”等正向关键词的相似度）、**执行频率**（该动作被执行的次数）、**子状态探索潜力**（执行该动作后跳转页面的探索价值）。
        *   **动态动作空间处理**：由于候选动作随页面变化，WebCQ 将**状态向量与每个候选动作向量拼接**，然后输入同一个DQN网络来评估各自的Q值，从而解决动态动作空间问题。
    *   **QTRAN驱动的集中式训练**：
        *   **引入QTRAN**：QTRAN是一种先进的合作式MARL算法，它能学习一个联合Q函数，并分解为各个智能体的个体Q函数，无需像QMIX那样强加单调性约束，能更好地捕捉智能体间的复杂交互。
        *   **联合状态表示**：由于无全局状态，WebCQ将所有智能体的局部状态向量拼接起来，形成联合状态 `s = (s_1; s_2; ...; s_N)`，供联合Q函数使用。
        *   **轻量级同步机制**：解决QTRAN默认同步执行与Web测试异步、不同步的环境的冲突。每个智能体独立探索，各自完成至少一个动作后，才触发一次集中式训练，避免了慢速智能体拖慢整体的阻塞问题。
        *   **混合奖励函数**：设计了包含三个部分的混合奖励：**页面相似度奖励**（鼓励探索新页面）、**执行频率惩罚**（鼓励执行不重复的动作）、**时间补偿**（随时间增加奖励，鼓励长期探索）。
