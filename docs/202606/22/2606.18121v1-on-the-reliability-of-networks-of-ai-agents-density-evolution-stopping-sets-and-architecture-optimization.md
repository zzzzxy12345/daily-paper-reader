## 摘要
现代AI系统越来越倾向于通过多个不完美智能体的协作（而非单个模型调用）来解决问题：一些智能体提出解决方案的片段，另一些进行验证，然后将结果合并。这类系统通常优于任何单个模型，但其成功或失败的原因往往不明确。我们将此类系统建模为在稀疏图上的消息传递，该结构是低密度奇偶校验（LDPC）码的基础，并将编码理论的密度演化机制扩展到这一更丰富的场景中。在模型中，任务是一组耦合的二元子主张，智能体架构是一个稀疏的、角色类型化的因子图，其中校验节点是带有噪声的布尔验证器节点，每个节点计算其所触及子主张的局部布尔函数。三种不同的失败模式（智能体弃权、验证器返回无可用输出、智能体间消息丢失）均被建模为擦除，当智能体交换集值消息时传播。校验智能体通过一个单一的逻辑强制规则组合这些消息，该规则专门针对XOR、AND、OR、蕴含和Horn约束。这不仅仅是LDPC理论的重新标签：验证器函数是非线性的且数值不对称，三种失败模式不能简化为单一有效信道，因此需要新的阈值、有限长度和反向结果，而非直接重用奇偶校验密度演化。我们证明了一个密度演化定理，该定理预测了随机角色类型化架构上未解决子主张的渐近比例，并扩展到确定性的、局部树状图序列。XOR情况恢复了二进制擦除信道（BEC）上的经典LDPC递推；AND情况揭示了正负验证器证书之间的不对称性。

## Abstract
Modern AI systems increasingly solve a task not with a single model call but with several imperfect agents working together: some propose pieces of a solution, others verify them, and the results are combined. These systems often outperform any single model, yet it is rarely clear why they succeed or when they will fail. We model such a system as message passing on a sparse graph, the structure that underlies low-density parity-check (LDPC) codes, and extend the density-evolution machinery of coding theory to this richer setting. In our model a task is a set of coupled binary subclaims, and an agent architecture is a sparse, role-typed factor graph whose check nodes are noisy Boolean verifier nodes, each computing a local Boolean function of the subclaims it touches. Three distinct failure modes, all modeled as erasures (an agent abstaining, a verifier returning no usable output, and a message lost between two agents), propagate as the agents exchange set-valued messages. The check agents combine these messages by a single logical-forcing rule that specializes to XOR, AND, OR, implication, and Horn constraints. This is more than a relabeling of LDPC theory: the verifier functions are nonlinear and value-asymmetric, and the three failure modes do not reduce to a single effective channel, so they require new threshold, finite-length, and converse results rather than a direct reuse of parity-check density evolution. We prove a density-evolution theorem that predicts the asymptotic fraction of unresolved subclaims on random role-typed architectures, with an extension to deterministic, locally tree-like graph sequences. The XOR case recovers the classical LDPC recursion on the binary erasure channel (BEC); the AND case exposes an asymmetry between positive and negative verifier certificates.

## 2. 方法论：核心思想、关键技术细节与公式流程

### 2.1 核心思想
- 将任务分解为一组耦合的二元子声明（subclaims）\(X^*_i \in \{0,1\}\)，表示第 \(i\) 个子任务是否有效（例如某个证明步骤是否正确，某个测试是否通过）。
- 智能体网络由**变量节点**（持有子声明的智能体）和**校验节点**（本地布尔验证器，如AND、XOR、Horn函数）组成，构成稀疏二部因子图。
- 三种擦除故障模式：
  1. **变量侧擦除** \(\epsilon^V_r\)：角色 \(r\) 的变量智能体弃权（超时、无可靠输出）。
  2. **验证器侧擦除** \(\epsilon^C_s\)：角色 \(s\) 的验证器无可用输出（超时、依赖失败）。
  3. **推理信道擦除** \(1-\eta_{r,s}\)：从角色 \(r\) 到 \(s\) 的消息丢失（格式转换失败、解析错误）。
- 所有噪声在 \(t=0\) 一次注入，之后消息传递是确定性的。消息是集合 \(\mathcal{M} = \{\{0\},\{1\},U\}\)（\(U=\{0,1\}\) 表示未解决）。校验节点使用 **逻辑强制规则** 更新目标变量：\(\Gamma_{\theta,j}(z; M_{-j})\)，即给定观测输出 \(z\) 和其他输入候选集，返回唯一一致的值或 \(U\)。

### 2.2 关键技术细节
- **密度演化（DE）**：对于足够大的图，每个角色类型 \(\tau\) 的变量到校验消息的擦除概率 \(p^{(b)}_{\ell,\tau}\) （条件于目标值 \(b\)）遵循确定性递归：
  - 校验到变量：\(h^{(b)}_{\ell,(\theta,j)} = 1 - (1-\epsilon^C_{s_\theta})\eta_{s_\theta,r_{\theta,j}}\varphi^{(b)}_{\ell,\theta,j}\)，其中 \(\varphi^{(b)}\) 是逻辑强制成功的概率。
  - 变量到校验：\(p^{(b)}_{\ell+1,\tau} = \epsilon^V_{r(\tau)} \mathbb{E}\left[\prod_{\tau'\in T_r} (h^{(b)}_{\ell,\tau'})^{D^{(r),\tau,\text{ex}}_{\tau'}}\right]\)，期望关于过量度分布。
  - XOR特例：退化为经典LDPC-BEC递归（单值，无需条件于 \(b\)）。
  - AND特例：\(b=1\) 时强制概率为 \(\prod_{k\neq j}\beta_{r_{\theta,k}}\)，\(b=0\) 时依赖其他入边非擦除概率，揭示正负证书不对称性。
- **浓度定理**（Theorem 1）：对固定 \(L\)，经验位擦除率依概率收敛到DE预测，且满足指数尾部界（McDiarmid不等式）。
- **停止集**（Theorem 3）：定义证书停止集，证明确定性剥落解码器的终端未解决集是唯一最大证书停止集。XOR恢复经典BEC停止集条件；AND给出正/负证书条件。
- **分隔增强**（Theorem 4）：增加验证节点使得每个大小为 \(\leq k\) 的小停止集至少有一个变量被新增验证器强制，可消除所有小停止集。
- **架构优化**（Theorem 6）：在成本约束下最小化残余擦除率，通过伴随灵敏度方程（反向传播）和KKT条件得到影子价格，识别最值得改进的旋钮（变量侧、验证器侧或信道）。
- **逆定理**（Theorem 7）：在 \(T\) 轮可靠消息传递协议类中，逻辑强制解码器是渐近最优的（无法以更小的未解决率超越DE预测）。

### 2.3 算法流程（文字说明）
1. 给定任务图（角色类型、度分布、验证器模板）和擦除参数。
2. 运行DE递归（公式 (24)-(28)）得到渐近残余擦除率 \(P^{(L)}_{DE}\)。

## 6. 主要结论与发现

1. **DE递归有效**：对于稀疏角色类型化因子图上的逻辑强制解码器，位擦除率依概率收敛到DE预测，有限长度偏差随 \(n^{-1/2}\) 衰减（Theorem 1）。
2. **三层不可互换**（Proposition 2）：变量侧擦除、验证器侧擦除、信道擦除不能简化为单一有效噪声，在参数雅可比中秩至少为2（多角色时秩至少为3，Proposition 3）。
3. **AND不对称性**：正证书（\(z=1\)）可同时强制所有输入为1，负证书（\(z=0\)）需要所有其他输入已知为1才能强制目标为0，导致 \(p^{(0)} > p^{(1)}\)（Proposition 1, 图4）。
4. **停止集表征**：证书停止集是终端未解决集的精确结构（Theorem 3）；XOR特例恢复经典BEC停止集；AND特例给出正/负条件。
5. **分隔增强消除小停止集**（Theorem 4, 5）：通过增加验证节点（或随机采样），可以高概率消除所有大小为 \(\leq k\) 的停止集。
6. **架构优化**（Theorem 6）：存在成本约束下的最优设计；伴随法可计算梯度；KKT条件给出影子价格。
7. **逻辑强制最优性**（Theorem 7）：在所有 \(T\) 轮可靠局部协议中，逻辑强制解码器渐近最优（无法被任何其他可靠协议超越）。
8. **短环导致错误底限**（图6）：短环密度增加时，经验残余从DE预测上升，说明局部树状性假设的失效代价。

## 7. 优点

- **理论创新**：首次将LDPC密度演化推广到非线性布尔验证器、角色类型化图和三重擦除，并在每个层面给出严格定理证明。
- **统一框架**：通过单一逻辑强制规则覆盖XOR、AND、OR、Horn等多种布尔原语，使不同智能体系统可以在同一理论下分析。
- **实用性导向**：提供了从系统日志校准参数（\(\epsilon^V, \epsilon^C, \eta\)）的协议，并给出架构优化策略（伴随梯度、KKT），有部署指导意义。
- **逆定理**（Theorem 7）证明逻辑强制在可靠协议类中的渐近最优性，增强理论信心。
- **实验验证充分性**：蒙特卡罗仿真与DE预测高度吻合，误差界与理论一致；短环实验展示了假设边界。

## 8. 不足与局限

- **实验覆盖有限**：仅验证了XOR和AND两种原语，未涉及OR、Horn、混合模板。未在大规模系统（如数千智能体）或真实多智能体工作流（如Hilbert、CodeR）上进行定量校准测试。
- **假设限制**：
  - **只能处理擦除**（可靠但可能缺失的消息），不能处理错误但自信的消息（需要BSC/吸收集扩展）。
  - **独立假设**：假设擦除在不同智能体间条件独立；实际中共享训练数据可能导致相关失败。
  - **图随机性假设**：DE定理基于配置模型随机图，但实际部署的任务图是确定性的。作者给出了确定性图局部弱收敛下的扩展（Theorem 8），但未验证实际非树结构（如小世界图）的适用性。
  - **一次噪声注入**：所有擦除在 \(t=0\) 确定，未建模每轮重新取样。
- **无实际系统对比**：在真实智能体系统上进行校准和预测的验证尚未进行，仅给出映射（Section XIII）而无数值结果。
- **计算资源未报告**：可能影响可重复性，但规模小影响不大。
- **优化非凸性**：架构优化问题非凸，KKT只给出局部最优，全局最优需枚举离散设计。
