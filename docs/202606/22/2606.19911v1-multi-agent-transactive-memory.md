## 摘要
具有不同能力的LLM智能体在多样任务上的分散部署，激发了跨异构智能体群体的知识共享基础设施的需求。正如搜索引擎索引人类生成的产物以支持人类解决问题，检索系统可以组织智能体生成的产物，以便在智能体群体中重复使用。我们将检索增强生成——它展示了人类撰写的产物对单个智能体的价值——扩展到检索智能体生成的产物，以支持智能体群体。特别是，智能体轨迹编码了可重用的程序性知识，然而这些产物通常在使用一次后被丢弃，或仅由生成智能体保留，迫使新实例化的智能体反复重新发现已有的解决方案。我们提出了多智能体交互记忆（MATM），这是一个用于群体级别存储和检索智能体生成轨迹的框架，其中生产智能体将轨迹贡献给共享存储库，消费智能体检索它们以改进任务执行。我们专注于交互环境（ALFWorld和WebArena），其中轨迹较长且编码了特别丰富的程序性结构。我们的实验表明，从MATM中检索轨迹可以改善下游任务性能，并减少交互步数，而无需协调或联合训练。这些结果将MATM定位为开放智能体生态系统中群体级别经验共享的设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：随着异构LLM智能体被部署到日益多样的领域，有必要建立支持分散智能体群体的基础设施。当前，智能体在执行任务过程中会生成丰富的轨迹（trajectories），其中编码了可重用的程序性知识。然而，这些轨迹通常在被使用一次后就被丢弃，或仅由产生它的智能体保留，导致新实例化的智能体不得不反复重新发现已有解决方案。
- **整体含义**：本文提出**多智能体交互记忆（Multi-Agent Transactive Memory, MATM）**，这是一个群体级别的存储和检索智能体生成轨迹的框架，借鉴了人类群体中“交互记忆”的概念（知识分布共享、协调检索）。MATM让“生产者”智能体将轨迹贡献到共享仓库，“消费者”智能体检索这些轨迹以改进自身任务执行，而无需协调或联合训练。这项工作旨在为开放智能体生态系统提供一种可扩展的群体经验共享设计模式。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将智能体产生的轨迹视为可检索的群体级记忆资源，建立一个共享索引。
- 采用**状态条件键值索引**：以当前任务描述和最近若干步交互历史作为**键（key）**，接下来的若干步动作-观察段作为**值（value）**（即检索得到的文档chunk）。
- 检索分为两个阶段：**单阶段密集检索**（使用E5-Base嵌入模型）和**学习排序（Learning-to-Rank, LTR）重排序**，后者利用特征工程预测轨迹对消费智能体的边际效用。

### 关键技术细节
- **索引构建**：预填充阶段从公开轨迹和数据生成收集初始索引；增量更新阶段，系统处理训练问题，让生产者智能体先无检索运行获得基线，然后在随机分支点检索top-K候选，评估每个候选的边际效用（检索后任务得分 - 基线得分），并将成功轨迹加入索引。
- **重排序模型（LTRT）**：使用44维特征，涵盖生产者元数据、消费者元数据、检索分数、查询特征、轨迹特征和查询-轨迹交互特征。训练标签为边际效用。比较了点式FFN、成对LambdaMART和SVMRank三种重排序器。
- **推理流程**：消费者智能体配备一个“RetrievalPlanner”LLM，在每个交互步骤决定是否需要发起检索调用，只有当前状态与已检索轨迹不相似时才触发检索。检索深度为20，经过重排序后取top-1轨迹chunk作为上下文。

### 算法流程（文字说明）
1. 预填充：收集公开轨迹，分割为l=5步的chunk，用E5-Base编码后插入索引D0。
2. 增量更新（P个分区）：
   - 每个分区，对每个训练问题，分配生产者智能体并运行无检索基线获得轨迹Tbase和得分sbase。
   - 随机选取T个分支点，在每个分支点检索top-K候选，对每个采样的rank j，让智能体基于该候选进行一次完成（one-shot rollout），记录得分s(j)_t，并计算边际效用s(j)_t - sbase。
   - 将成功轨迹加入缓冲区，分区处理完后将缓冲区内所有轨迹分块、编码、添加至索引。
3. LTRT训练：用收集的（query, document, marginal utility）三元组训练重排序器。
4. 推理：消费者智能体每步通过RetrievalPlanner决定是否检索，若检索则索引返回top-20，重排序器选出最佳chunk作为上下文指导当前动作生成。
