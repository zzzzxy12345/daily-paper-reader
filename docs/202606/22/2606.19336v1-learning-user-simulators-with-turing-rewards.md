## 摘要
在交互式环境中学习模拟人类用户，可以推进代理助手的训练、个性化系统的评估、社会科学的研究等。现有方法通常通过训练大型语言模型（LLM）来匹配单一的真实响应，要么通过最大化对数概率，要么使用相似性奖励。我们提出{Turing-RL}：一种基于图灵测试的强化学习方法，用于训练用户模拟器模型。{Turing-RL}使用一个判别性的图灵奖励，通过LLM评判器来评估生成的响应与真实用户历史背景下响应的不可区分程度，用户模拟器LLM学习在这些奖励下产生与用户可能说过的话不可区分的响应。在两个不同的领域——对话聊天和Reddit论坛讨论——我们发现{Turing-RL}在LLM和人类评估指标上均持续优于基线方法。我们的研究表明，优化不可区分性而非响应匹配是学习用户模拟器的有效方法。

## Abstract
Learning to simulate human users in interactive settings could advance the training of agent assistants, evaluation of personalization systems, research in the social sciences, and more. Existing approaches generally do so by training a large language model (LLM) to match a single ground truth response, either by maximizing the log probability or by using a similarity reward. We instead propose {Turing-RL}: a Turing-Test-based reinforcement learning approach for training user simulator models. {Turing-RL} uses a discriminative Turing reward with an LLM judge to score how indistinguishable a generated response is from the real user's given the user's history, and the user simulator LLM learns to produce responses indistinguishable from what the user could have said with such rewards. Across two different domains--conversational chat and Reddit forum discussion--we find that {Turing-RL} consistently outperforms baseline methods on both LLM and human evaluation metrics. Our study suggests that optimizing for indistinguishability, rather than response matching, is effective for learning user simulators.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：用户模拟器对于训练智能助理、评估个性化系统、社会科学研究等至关重要。现有方法通常训练大语言模型（LLM）直接匹配单一真实用户响应（通过最大化对数概率或使用相似性奖励），但这种做法忽略了“同一用户在相同上下文下可以说出多种合理响应”的事实。理想模拟器应能生成与真实用户不可区分的响应，而非仅仅复制某一条真实回复。
- **整体含义**：论文提出一种基于图灵测试的强化学习方法（Turing-RL），将“不可区分性”直接作为训练信号，从而学习更逼真的用户模拟器。这为构建更自然、更有用的交互式模拟系统提供了新范式。

## 2. 方法论：核心思想、关键技术细节与流程

- **核心思想**：利用判别性图灵奖励（Turing Reward）训练用户模拟器。该奖励由LLM评判器给出，衡量模型生成的响应与真实用户响应在给定用户历史上下文下的不可区分程度。优化目标不是让模型输出与真实响应内容重叠，而是让输出听起来像真人所说。
- **关键技术细节**：
    - **问题形式化**：给定当前会话上下文 \(x\) 和用户表示 \(u\)（由用户历史行为 \(h\) 与诱导人格 \(ρ\) 组成），学习策略 \(p_θ(y|x,u)\) 以生成不可区分的响应 \(y\)。
    - **图灵奖励**：LLM评判器（使用Qwen3.5-397B-A17B）在1-7 Likert量表上对模型响应与真实响应进行对抗性比较，得分越高表示模型响应更似真人。奖励值经截断（cap at 5）并归一化至[0,1]以防止奖励破解。公式：\(r_{turing}(y, y^\star) = \frac{\min\{s(y, y^\star), 5\} - 1}{6}\)。
    - **优化算法**：使用GRPO（Group Relative Policy Optimization）进行强化学习训练。每组采样4个候选响应，计算组内相对优势，并加入KL散度惩罚（β=1e-3）和长度惩罚（惩罚偏离真实响应长度的输出）。
    - **SFT预热**：先在有监督微调阶段，使用CoT（Chain-of-Thought）推理轨迹增强训练数据。推理轨迹由教师模型生成，解释可能引导用户写出真实响应的原因，并与真实响应拼接作为SFT目标。
    - **用户表示**：默认使用用户历史 \(h\) 与诱导人格 \(ρ\)；诱导人格由GPT-5.4 nano从历史块中总结出五个字段（价值观、口头禅、表达风格、长度习惯、背景）。
- **算法流程**：
    1. 收集用户历史 \(h\) 与当前上下文 \(x\)。
    2. 对每个用户诱导人格 \(ρ = f_{LLM}(h)\)。
    3. SFT阶段：用CoT增强训练模拟器。
    4. RL阶段：从SFT模型初始化；对每个训练样本，模型生成4个候选响应；LLM评判器计算每个候选响应的图灵奖励；GRPO更新策略。

## 3. 实验设计

- **数据集与场景**：
    - **多轮对话 (Chat)**：PRISM Alignment Dataset（含1,500名参与者，选1,288名至少6次对话的用户，保留128名用户共880条测试样本）。
    - **Reddit论坛讨论**：ConvoKit子主题语料库（选14个子版块，1,282名至少8个帖子的用户，保留r/tifu和r/worldnews共102名用户267条测试样本）。
- **基准对比**：
    - **Sim-RL**：使用响应相似性奖励（Wu et al., 2026），由LLM评判语义对齐程度。
    - **Logprob-RL**：使用真实响应的对数概率作为奖励（Gandhi et al., 2026）。
    - **SFT-Init**：仅经过SFT预热的模型。
    - **未训练基线**：原始Qwen3-8B、Qwen3.5-397B-A17B、GPT-5。
    - 所有RL方法均从同一SFT检查点初始化，使用相同历史/人格信息、相同GRPO训练配置。
- **评估指标**：
    - **LLM评判**：使用Claude Sonnet 4.6作为评判器，评估三个维度：
        - 图灵可区分性（1-7分，越高越像真人）。
        - 响应相似性（0-1，越高越匹配真实响应内容）。
        - 上下文与用户特异性（0-1，越高越与当前交互和用户兼容）。
    - **人类评估**：在Prolific上招募300多名参与者进行二选一图灵测试，每个条件收集600个判断，报告模型胜率（>0.5表示模型响应超过机会水平）。

## 6. 主要结论与发现

1. **Turing-RL显著优于所有基线**：在Chat和Reddit两个领域，Turing-RL的图灵可区分性得分均最高，差距在Chat上尤为明显。
2. **优化不可区分性不牺牲内容匹配**：Turing-RL的响应相似性得分与Sim-RL相当，表明生成更人性化的同时没有丢失对真实内容的对齐。
3. **内容匹配不等于人性化**：Sim-RL虽然提升了相似性，但图灵可区分性得分与SFT-Init相近，说明仅匹配内容不足以提升逼真度。
4. **人类与LLM评判趋势一致**：人类评估中Turing-RL在Chat上胜率最高（0.57），在Reddit上显著优于SFT-Init（p=0.0095），与LLM自动评判结果吻合。
5. **用户表示影响有限但存在**：历史+人格组合在图灵得分上最稳健；仅人格即可达到相似的不可区分性，但内容相似性较差。
6. **强模型（GPT-5、Qwen3.5-397B）不一定人性化**：这些模型常生成过于冗长、修饰过的“助理式”回复，反而降低图灵得分。

## 7. 优点

- **创新奖励设计**：将图灵测试理念直接引入强化学习训练信号，而非使用传统的生成-匹配范式，思路新颖且有效。
- **实验全面**：覆盖两个差异较大的交互领域，同时包含自动评估与人工评估，结果可信度高。
- **公平比较**：强调所有RL基线共享相同的SFT初始化、GRPO训练配置、历史/人格输入，确保对比公平。
- **深入分析**：做了详细消融（用户表示、人格来源）、定性展示、训练动态曲线，并讨论了人机评判差异与潜在风险。
- **实用价值**：证明了较小模型（Qwen3-8B）通过合适的RL训练可超越更强基础模型，为资源受限场景提供可行方案。

## 8. 不足与局限

- **领域覆盖有限**：只测试了开放式聊天和论坛讨论，未包括任务导向型对话、协商、协作解决问题等其他交互类型，泛化性待验证。

- **未评估推理过程对齐**：论文只关注输出文本的人类相似性，未检查模型内部推理过程是否与人类认知过程一致。
