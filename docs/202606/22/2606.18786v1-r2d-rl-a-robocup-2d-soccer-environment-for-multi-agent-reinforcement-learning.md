## 摘要
机器人足球是多智能体强化学习的一个具有挑战性的试验平台，因为它结合了部分可观测性、合作与对抗交互、稀疏奖励以及长时间跨度的战术行为。RoboCup 2D足球仿真（RCSS2D）提供了一个成熟的机器人足球平台，但其面向竞争的服务器-客户端架构难以直接与现代基于Python的MARL工作流结合使用。我们引入了R2D-RL，这是一个强化学习环境，通过共享内存通信和周期级同步将RCSS2D和基于HELIOS的玩家客户端连接到Python MARL接口。R2D-RL支持全场和基于场景的训练，具有可配置的对手、基本离散和混合参数化动作空间、动作掩码、基于期望控球值（EPV）的奖励塑造以及并行执行。我们提供了前场进球场景和一个11对11的全场基准，以及基线结果。

## Abstract
Robot soccer is a challenging testbed for multi-agent reinforcement learning because it combines partial observability, cooperative and adversarial interaction, sparse rewards, and long-horizon tactical behavior. RoboCup 2D Soccer Simulation (RCSS2D) provides a mature robot-soccer platform, but its competition-oriented server-client architecture is difficult to use directly with modern Python-based MARL workflows. We introduce R2D-RL, a reinforcement learning environment that connects RCSS2D and HELIOS-based player clients to a Python MARL interface through shared-memory communication and cycle-level synchronization. R2D-RL supports full-field and scenario-based training with configurable opponents, Base discrete and Hybrid parameterized action spaces, action masks, expected possession value (EPV)-based reward shaping, and parallel execution. We provide front-goal scenarios and an 11-vs-11 full-field benchmark, together with baseline results.

## 2. 论文提出的方法论

- **核心思想**：将 RCSS2D 的竞赛流程转换为步进式强化学习交互。通过共享内存的序列计数器协议（`obs_seq`、`act_seq`、`done_seq`）同步各玩家客户端与 Python 环境，确保每个环境步对应一个 `play_on` 模组下的服务器周期，动作执行与状态转换一一对应。
- **关键技术细节**：
  - **通信架构**：修改后的 HELIOS 玩家、教练和训练器客户端通过共享内存缓冲区与 Python 环境交换观测、动作掩码、集中状态和重置请求。
  - **动作空间**：
    - **Base 离散空间**：19 个高层语义动作（射门、传球、盘带、移动等），由 HELIOS 内部模块执行底层命令。
    - **Hybrid 参数化空间**：6 个离散命令 ID（turn、dash、kick、catch、HELIOS Fallback、empty），每个命令附有连续参数（角度、力度等）。
  - **动作掩码**：基于 HELIOS 可执行性检查（如射门仅当球可踢且满足条件时启用），Base 空间中射门启用时屏蔽其他动作；Hybrid 空间按玩家状态（是否控球、是否冻结）屏蔽不合法命令。
  - **奖励设计**：标准得分奖励（±1） + 可选 `MaxEPV` 塑形奖励。EPV 基于 32×50 网格，仅当控球队成员可踢球时更新最大 EPV，奖励正向增量，射门得分时补足至网格最大 EPV。
  - **场景重置**：使用训练器客户端直接移动球和玩家到指定状态（位置、方向、速度），避免重启服务器，支持内建基准和用户自定义初始状态。
  - **并行执行**：多环境实例通过 SLURM 分配 CPU 资源并行采样，提高吞吐量。

- **基准方法**：

## 6. 论文的主要结论与发现

- **环境可用性**：R2D-RL 支持从简单控球射门到 11v11 全场的渐进式训练，并行采样可达 34–41M 步/天（14 实例）。
- **算法对比**：MAPPO 在多攻击者场景和全场基准中表现最稳定；QMIX 在某些场景（如 Passing Lane、Blocked Shot 带掩码）有效，但种子方差大；ParaDQN 仅在简单零星场景有效，多攻击者场景及全场几乎为零。
