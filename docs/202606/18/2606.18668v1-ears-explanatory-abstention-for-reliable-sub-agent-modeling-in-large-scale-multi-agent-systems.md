## 摘要
在大型企业环境中，集中式多智能体系统（MAS）日益普及，其中协调器将用户请求委托给轻量级、领域专门化的子代理。虽然这种架构提高了模块化、可扩展性和成本效率，但其可靠性不仅依赖于准确的路由，还依赖于子代理根据能力约束校准其响应的能力。特别是，基于较小微调模型的子代理往往难以进行此类校准，导致它们对模糊、未明确说明、错误路由或不支持的请求过度回答，产生幻觉输出，而非可操作的反馈。为应对这一挑战，我们提出了EARS（用于可靠子代理建模的解释性放弃），这是一个面向生产的框架，将子代理放弃重新定义为一种代理间通信协议：子代理不仅放弃，还会向协调器暴露一个可操作的失败状态。EARS使用一组校准的LLM作为评判模型来策展人机交互数据，在子代理失败模式的分类法下生成结构化的放弃标签和理由。这些数据用于微调子代理，以检测失败条件并返回理由，供协调器级澄清、重新路由或回退。我们在一个支持企业商业智能工作流程的大规模生产电商助手中评估了EARS。EARS将整体响应通过率从68.5%提高到78.9%，表明子代理侧的解释性放弃提高了MAS的可靠性。

## Abstract
In large-scale enterprise settings, centralized multi-agent systems (MAS) are increasingly adopted, in which a coordinator delegates user requests to lightweight, domain-specialized sub-agents. While this architecture improves modularity, scalability, and cost efficiency, its reliability depends not only on accurate routing but also on sub-agents' ability to calibrate their responses to capability constraints. In particular, sub-agents built on smaller fine-tuned models often struggle with such calibration, leading them to over-answer ambiguous, underspecified, misrouted, or unsupported requests and produce hallucinated outputs instead of actionable feedback. To address this challenge, we present EARS (Explanatory Abstention for Reliable Sub-Agent Modeling), a production-oriented framework that reframes sub-agent abstention as an inter-agent communication protocol: a sub-agent does not merely abstain, but exposes an actionable failure state to the coordinator. EARS curates human-agent interaction data using an ensemble of calibrated LLM-as-a-Judge models, producing structured abstention labels and rationales under a taxonomy of sub-agent failure modes. These data are used to fine-tune sub-agents to detect failure conditions and return rationales for coordinator-level clarification, rerouting, or fallback. We evaluate EARS in a large-scale production e-commerce assistant supporting enterprise business intelligence workflows. EARS improves the overall response pass rate from 68.5% to 78.9%, demonstrating that sub-agent-side explanatory abstention improves MAS reliability.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：在大型企业级集中式多智能体系统（MAS）中，协调器将用户请求委托给轻量级、领域专精的子代理（通常基于小型微调模型）。子代理往往难以正确校准自身能力边界，倾向于对模糊、未指定、错误路由或不支持的请求过度回答（over-answer），产生幻觉输出（hallucination），而非提供可操作的失败反馈。这导致系统可靠性下降。
- **动机**：现有工作主要关注协调器层的路由优化（如RIRS、MasRouter、成本感知编排），但忽视了子代理执行失败及由此引发的通信断裂。MAS失效分析（Cemri et al., 2025）表明，代理间信息流中断是系统级故障的核心原因之一。子代理返回的内容不仅是中间负载，更是下游恢复的通信信号。
- **整体含义**：EARS提出将子代理的“放弃”（abstention）重新定义为一种面向协调器的**可操作通信协议**，使放弃从最终用户拒绝转变为代理间协调信号，从而提升MAS整体可靠性。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：解释性放弃（Explanatory Abstention）。子代理在无法正确执行时，不仅拒绝回答，还返回**结构化的失败状态**（放弃类别+理由），供协调器进行澄清、重路由或回退。
- **关键技术细节**：
  1. **放弃分类法**：设计四类互斥失败模式：
     - 模糊查询（Ambiguous Query）
     - 输入不足（Insufficient Input）
     - 能力缺失（Missing Capability）
     - 错误路由（Misrouting）
     每类附带具体理由（如模糊源、缺失信息、不支持功能、应转交的代理）。
  2. **人工标注种子集**：采用独立双标注+仲裁方式，对少量人机交互数据标注放弃类别和理由，获得600个可解+602个放弃实例（共1202条）。
  3. **LLM-as-a-Judge校准**：
     - 基于种子集，迭代优化评判模型的提示指令（人工分析错误模式+修改），直至分类准确率超过阈值（如80%）。
     - 使用多模型集成策略：从三个模型家族（GPT-5、o3、Claude-sonnet-4.5、Gemini-2.5-flash）中挑选四个模型，仅当所有模型一致同意时接受标签（四模型共识），以保证高精度。
  4. **数据策展飞轮**：应用校准后的评判器对大规模生产交互数据（13.8K样本）进行自动标注，生成放弃标签和理由。
  5. **子代理微调**：在Qwen3-32B-Instruct上进行全参数监督微调（SFT），训练实例为用户查询 + 子代理上下文 + 期望输出（可解时输出DSL查询，放弃时输出结构化放弃响应）。微调后的子代理重新部署，收集新交互，形成在线改进飞轮。

- **问题定义新颖**：将子代理放弃从“用户拒绝”重新定义为“代理间通信协议”，填补了MAS可靠性研究中子代理侧反馈信号的空缺。
- **实用导向设计**：面向生产系统，结合在线数据飞轮，可实现迭代改进。
- **数据质量控制**：采用人工标注种子集+多模型共识策展，避免低质量标签污染训练数据。
- **真实系统验证**：在支持数百万客户的大型电商平台进行实验和人类评估，结果具有现实意义。

## 8. 不足与局限
- **领域单一**：仅评估了电商BI子代理，在金融、教育、医疗等领域的泛化性未验证。
- **依赖人工标注与校准**：LLM-as-a-Judge校准需要人类分析错误模式并迭代修改提示，成本高且难以扩展；虽提及自动校准方法，但未采用。
- **放弃分类混淆**：实验显示模糊查询与输入不足类别间存在显著混淆（如表中o3模型在Ambiguous Query上仅33.3%准确率），分类界限可能不够清晰。
- **未与现有放弃方法对比**：虽然那些方法面向不同设定，但作为MAS可靠性提升技术，缺少直接基准（如Abstain-R1、LatentRefusal）会削弱说服力。
- **实验设计缺陷**：基线训练数据量可能少于EARS（基线仅可解查询，EARS额外包含放弃样本），未控制训练数据规模一致；消融实验中“仅人工标注”设置混合了原始生产数据，引入标签不一致性，其性能低于EARS的策展数据，但更干净的对照应是将人工标注数据直接替换原始数据。
- **协调器侧优化未探索**：EARS聚焦子代理端，但协调器如何利用放弃信号（如是否采纳重路由建议）未做优化，留作未来工作。
- **资源成本**：四模型共识策展成本高达单模型的20.4倍，实际部署中可能需权衡精度与成本。
