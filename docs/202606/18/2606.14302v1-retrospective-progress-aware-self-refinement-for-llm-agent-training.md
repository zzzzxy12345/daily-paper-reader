## 摘要
基于强化学习训练的LLM智能体优化了逐步动作预测，但缺乏对任务进度的元认知意识，这造成了一个阻碍长期扩展的鸿沟。一项试点研究表明，在线进度提示会损害性能，而回顾性演示则有所帮助，但这种能力无法仅从结果奖励训练中涌现。我们提出了RePro（回顾性进度感知训练），这是一种通过“先行动后反思”的推演范式训练智能体自我生成进度信号的框架：智能体在线执行动作，然后根据已完成的轨迹和已知结果回顾性地重新评估其逐步进度。RePro首先通过回顾性热身（Retrospection Warmup）从最少的外部演示中教授反思格式，然后通过RePro-PO使用复合奖励进一步训练，该奖励产生自我生成的信号，无需持续的外部监督。在WebShop、ALFWorld和Sokoban上的实验表明，RePro提升了Qwen系列的性能，成功率绝对提升高达12%。

## Abstract
LLM-based agents trained with reinforcement learning optimize step-wise action prediction but lack metacognitive awareness of task progress, inducing a gap that hinders long-horizon scaling. A pilot study reveals that online progress prompting hurts performance while retrospective demonstrations help, yet this capability cannot emerge from outcome-reward training alone. We present RePro, Retrospective Progress-Aware Training, a framework that trains agents to self-generate progress signals via a forward-then-reflect rollout paradigm: the agent executes actions online, then retrospectively reassesses its step-wise progress given the completed trajectory and known outcome. RePro initializes with a Retrospection Warmup that teaches reflection format from minimal external demonstrations, then further trains through RePro-PO with a composite reward that produces self-generated signals without continuous external supervision. Experiments on WebShop, ALFWorld, and Sokoban show that RePro enhances the Qwen family's performance, with up to $12\%$ absolute success rate gains.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体在长期任务（long-horizon tasks）中仅通过强化学习优化逐步动作预测，但缺乏对任务进度的元认知意识（progress awareness），即“已完成什么、还差什么、当前轨迹是否在正确轨道上”的持续感知。这种缺失导致智能体无法合理分配多步探索中的精力，限制了长期任务的成功率。
- **背景**：现有 RL 训练方法（如 GRPO、GiGPO）依赖稀疏结果奖励，难以提供密集的中间反馈；过程奖励方法需要额外标注或环境特定设计。作者通过试点研究揭示了一个关键不对称：**在线进度提示会降低性能（平均下降8.6%），而回顾性进度演示能提升性能（平均提升7.9%），且该能力无法从结果奖励训练中自动涌现**。这证明进度感知需要专门训练。

## 2. 方法论

### 2.1 核心思想：前向-反思（Forward-then-Reflect）范式

智能体在任务执行过程中同时输出**在线进度估计**和动作；任务完成后，根据完整轨迹和已知结果**回顾性地重新评估**每一步的实际完成进度。通过将回顾性进度信号整合进 RL 奖励，使智能体学会自我生成进度感知。

### 2.2 关键技术细节

- **前向执行**：每一步，智能体生成在线进度估计 `~(p_t)` 和动作 `a_t`，条件为历史。
- **回顾性反思**：轨迹结束后，使用外部模型（DeepSeek-V4）或自身策略生成回顾性进度 `(p_1,...,p_T)`，条件为完整轨迹和结果。成功轨迹的最后一步强制为100%，第一步为0。
- **第一阶段：Retrospection Warmup（回顾性预热）**  
  用成功轨迹的外部 LLM 演示构建标注数据集，进行监督微调（SFT），让智能体学习前向-反思的格式。损失函数包含前向预测损失和回顾性反思损失。
- **第二阶段：RePro-PO（进度感知策略优化）**  
  复合奖励函数：
  - 环境奖励 `r_env`（稀疏，仅终点非零）
  - 进度塑造奖励 `r_p(t) = β·(p_{t+1} - p_t)`，鼓励进度递增
  - 在线-回顾对齐奖励 `r_align(t) = (1/T)·α·max(0,1-|p_t - ~p_t|)`
  - 格式正则化奖励 `r_format`，惩罚边界违规（成功时 p1≠0 或 pT≠1）
- **优势函数**：采用层次化优势（episode-level 和 step-level），步骤优势基于同锚状态（相同观测）的折扣收益对比，使用 clipped PPO 优化。

## 6. 主要结论与发现

- **RePro 显著优于所有基线**：在 WebShop 上，绝对成功率提升 +8.98%（1.5B）、+11.57%（3B）、+5.82%（7B）。在 ALFWorld 上，1.5B 提升 +11.72 SR，7B 提升 +4.69 SR。Sokoban 上 4B 提升 +3.12 SR。
- **回顾性预热必不可少**：仅做 Warmup 后不加 RL 训练，1.5B 和 3B 模型崩溃，7B 下降 23.05%，证明格式学习不等于有效进度感知。
- **进度质量得到本质提升**：RePro 的中间辨别力（IntermDisc）远高于基线（如 3B 模型从 −1.07 升至 31.62），成功/失败轨迹的进度动态显著分化，且失败轨迹出现更多合理的下降模式，表明智能体形成了更有意义的元认知。
- **进度感知可以自持续**：无需外部奖励模型，仅凭任务结果和自身回顾性信号即可学习有效的进度感知。

## 7. 优点

1. **创新性**：首次提出通过回顾性进展信号训练 LLM 智能体的元认知能力，解决了“在线无法生成但回顾有用”的不对称问题。
2. **自监督性**：仅需少量外部演示（初始预热），后续可自我生成信号，无需持续人工标注或额外奖励模型，实用性强。
3. **复合奖励设计合理**：结合进度塑造、在线-回顾对齐、格式正则化，有效利用了回顾性信息，且层次化优势函数协同了全局和局部信用分配。
4. **实验验证充分**：跨模型尺度、跨环境验证泛化性，并深入分析了进度预测质量，证明了方法对元认知的实质改善。

## 8. 不足与局限

1. **进度表示形式受限**：当前仅用 verbalized 数字（0-100%）表示进度，未探索潜在状态或隐藏向量监督等替代形式，可能限制了表达能力。
2. **环境覆盖有限**：仅测试了 WebShop、ALFWorld、Sokoban，未涉及更复杂的 GUI 自动化、开放式具身交互或命令行环境，泛化性有待验证。
3. **缺少地面真值标签**：进度本身无客观标准，依赖回顾性标注（来自强模型或自身），可能存在偏差。未来需引入人类标注或强 LLM 评估进一步校准。
4. **算力信息缺失**：未报告 GPU 类型、数量、训练总时长，使得方法复现和能耗评估困难。
5. **Sokoban 上的改进较小**：可能因该环境已有部分过程奖励（每推一个箱子成功就给奖励），进度感知的边际收益减少，说明方法在完全稀疏奖励场景下优势更明显。
