## 摘要
网页导航要求代理遵循自然语言目标、与网页交互并生成准确答案。尽管近期进展借助了视觉语言模型和强化学习，现有方法仍因奖励错位和错误传播而面临单步脆弱性问题。为解决奖励纠缠，我们设计了动态双策略优化（DDPO），该策略在导航优先模式（用于探索）与答案优先模式（用于问答）之间动态切换，以缓解奖励冲突。为校准单步错误，我们提出置信度引导的自适应导航反思机制（CANR），该机制估计每步置信度，仅在必要时触发反思，并利用对比奖励鼓励自我纠正，从而校准单步不准确性。基于上述主要组件，我们最终开发了StepGuard——一种通过单步校准守护网页导航的新框架。实验表明，我们的方法显著提升了导航与答案准确性，在标准网页导航基准上达到了新的最先进水平。

## Abstract
Web navigation requires agents to follow natural language goals, interact with web pages, and produce accurate answers. While recent advances leverage vision-language models and reinforcement learning, existing methods still suffer from single-step fragility due to reward misalignment and error propagation. To tackle the reward entanglement, we design Dynamic Dual-Policy Optimization (DDPO), which dynamically switches between a navigation-first mode for exploration and an answer-first mode for question-answering to mitigate reward conflict. To calibrate the single-step error, we propose Confidence-Guided Adaptive Navigation Reflection (CANR), a mechanism that estimates per-step confidence, triggers reflection only when necessary, and uses contrastive rewards to encourage self-correction to calibrate the single-step inaccuracy. With the above as the main components, we finally develop our StepGuard, a new framework of Guarding Web Navigation via Single-Step Calibration. Experiments demonstrate that our approach significantly improves navigation and answer accuracy, setting new state-of-the-art performance on standard web navigation benchmarks.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有网页导航（Web Navigation）智能体在单步决策上存在脆弱性，主要原因有二：
  - **奖励错配**：导航任务中的探索性奖励（鼓励继续浏览）与问答任务中的终止性奖励（鼓励尽快生成答案）相互冲突，导致策略优化不稳定。
  - **错误传播**：单步决策错误会沿轨迹累积，最终导致任务失败和答案错误。
- **重要性**：网页导航是构建自主网页助手、智能信息检索等应用的基础，提升其鲁棒性具有重要实际价值。
- **现有方法的不足**：尽管Vision-Language Models (VLMs)和强化学习（如GRPO）取得了进展，但未能从单步层面解决上述问题。

## 2. 论文提出的方法论

### 核心思想
通过**单步校准**（Single-Step Calibration）来缓解奖励冲突和纠正单步错误，具体包含两个关键技术。

### 2.1 动态双策略优化（DDPO，Dynamic Dual-Policy Optimization）
- **目标**：解耦导航与问答的奖励信号，避免梯度冲突。
- **机制**：将训练数据划分为导航子集（\(D_{nav}\)）和问答子集（\(D_{qa}\)），交替采用两种优化模式：
  - **导航优先模式**：奖励 = 格式奖励 \(R_f\) + 导航奖励 \(R_n\)（引导代理靠近目标页面，基于专家动作或页面距离度量）。
  - **答案优先模式**：奖励 = 格式奖励 \(R_f\) + 问答奖励 \(R_a\)（基于F1分数衡量答案准确性）。
- **公式**：\(R_{DDPO} = \begin{cases} R_a + R_f, & z_t \in D_{qa} \\ R_n + R_f, & z_t \in D_{nav} \end{cases}\)，其中 \(z_t\) 为单步训练样本的任务类型。
- **效果**：实现奖励解耦，稳定策略更新。

### 2.2 置信度引导的自适应导航反思（CANR，Confidence-Guided Adaptive Navigation Reflection）
- **目标**：检测并纠正单步决策错误，防止错误传播。
- **工作流程**：
  1. **置信度估计**：计算当前状态 \(s_t\) 下策略分布 \(\pi_\theta(\cdot|s_t)\) 与均匀分布 \(U(C_t)\) 的KL散度：\(\text{Conf}(s_t) = D_{KL}(\pi_\theta(\cdot|s_t) \parallel U(C_t))\)。高值表示置信度高，低值表示不确定。
  2. **概率性反射触发**：根据置信度计算触发反射的概率：\(P_{reflect}(s_t) = \exp(-\kappa \cdot \text{Conf}(s_t))\)，其中 \(\kappa\) 为敏感系数（默认为4）。低置信度时更可能触发反思。
  3. **反思与校准**：触发时，模型先生成初始响应 \(y_{init}\)，然后通过反思提示生成修正响应 \(y_{ref}\)，最终输出 \(y_{final} = y_{init} \oplus \text{reflection prompt} \oplus y_{ref}\)。
  4. **反思成功奖励**：引入 \(R_{rs} = I[R_n(s_t, a^{init}_t) < R_n(s_t, a^{ref}_t)]\)，仅当修正后的导航奖励更高时才给予正向奖励。最终反思奖励为 \(R_{ref} = R_f + R_n + R_{rs}\)。
- **整体训练**：使用GRPO算法优化，融合DDPO和CANR的奖励。

## 6. 论文的主要结论与发现

1. **性能提升**：StepGuard在WebVLN上达到SR 39.83%，超越SOTA模型WebVLN-Net（34.76%）约5%；在WebWalkerQA上，3B模型在Hard子集上达到25.38% SR，接近72B模型的25.83%，体现参数效率。
2. **DDPO与CANR互补**：两者均能独立提升性能，联合使用效果最佳。
3. **自适应反射优于固定策略**：CANR相比始终触发反射，在保证高性能的同时大幅减少轨迹长度和推理时间。
4. **置信度校准有效**：StepGuard能更好地区分正确与错误预测（置信度差距从0.06提升至0.17）。
5. **格式奖励至关重要**：移除 \(R_f\) 会导致无效动作率飙升（高达15.80%），说明轻量模型需要显式语法监督。

## 7. 优点

- **创新性**：
  - DDPO通过动态切换模式解耦奖励冲突，概念简单但有效。
  - CANR采用概率性触发机制，在不确定性高时引入反思，避免不必要的计算开销，且通过对比奖励强化正确修正。
- **实验全面性**：从主性能、消融、超参数、效率到定性分析，覆盖完整。
- **资源效率**：在3B小模型上取得与72B模型相当的性能，证明方法可缩小模型规模差距，降低部署成本。
- **泛化性**：框架兼容多模态和纯文本LLM，适用于多种视觉/文本网页场景。

## 8. 不足与局限

1. **依赖密集奖励工程**：尤其是格式奖励 \(R_f\) 对小型模型起到关键约束作用。若目标任务缺乏明确有效性信号或无法提供稠密监督，框架可能面临收敛困难。
2. **模型能力限制**：底层使用Qwen2.5-3B，其指令跟随能力较弱，可能在高复杂度任务中仍不如大模型，且对奖励设计更敏感。
3. **实验覆盖**：仅在两个单一类型数据集上评估，未涉及更多样化真实网页（如动态交互、JavaScript渲染等），泛化性需进一步验证。
4. **未讨论失败模式**：论文未详细分析反射失败的情况（例如反思后仍错误）以及对应原因。
5. **超参数依赖**：\(\kappa\) 和反射概率的设计需要调优，在不同任务中可能需要重新设定。
