## 摘要
三方市场中的配送为从世界反馈中进行强化学习提供了自然场景：决策通过延迟的操作结果（如配送速度、骑手利用率和商家拥堵程度）进行评估。我们介绍了DoorDash部署的一个强化学习系统，该系统利用延迟信号，在大规模食品配送市场中自适应地调整配送目标权重。该方案并未替代组合分配优化器，而是通过学习来自市场历史数据的店铺级策略，选择一个离散乘数，从而改变配送优化器在配送质量和批处理效率之间的权衡。这种接口使得在嘈杂、延迟和耦合的反馈下进行离线策略学习成为可能，同时保持了生产可行性约束和操作保障。我们利用集中式离线数据和分散的店铺级执行来训练共享价值函数，采用Double Q-learning目标和保守正则化器来减少分布外价值高估。在生产环境中的开关实验中，离线训练的策略提高了批处理效率，降低了骑手侧的时间成本，同时不影响面向客户的配送质量。结果展示了如何利用来自实时经济和物流系统的世界反馈，在线安全地调整决策策略。

## Abstract
Dispatch in three-sided marketplaces provides a natural setting for reinforcement learning from world feedback: decisions are evaluated by delayed operational outcomes such as delivery speed, courier utilization, and merchant congestion. We present a deployed reinforcement learning system at DoorDash that adapts dispatch objective weights in a large-scale food-delivery marketplace using delayed signals. Rather than replacing the combinatorial assignment optimizer, a store-level policy learned from logged marketplace data selects a discrete multiplier that shifts the dispatch optimizer's tradeoff between delivery quality and batching efficiency. This interface enables offline policy learning under noisy, delayed, and coupled feedback while preserving production feasibility constraints and operational safeguards. We train a shared value function using centralized offline data and decentralized store-level execution, with Double Q-learning targets and a conservative regularizer to reduce out-of-distribution value overestimation. In a production switchback experiment, the offline-trained policy increases batching and reduces courier-side time costs without degrading customer-facing delivery quality. Results illustrate how world feedback from a live economic and logistics system can be used to safely adapt decision policies online.

## 2. 方法论

### 核心思想
- 采用**两层决策架构**：
  - **内层**：已有的组合分配优化器，负责在约束下将订单分配给骑手。
  - **外层**：目标权重自适应代理（OWA-RL），是一个存储级别策略，学习从延迟的市场反馈中选择一个离散乘数，按比例缩放优化器中ASAP权重的基线值。
- 这种方式**不替换优化器**，而是通过一个低维控制接口（5个离散乘数）进行调优，保证生产可行性、延迟和可扩展性。

### 关键技术细节
- **公式化**：多智能体部分可观测马尔可夫决策过程（Dec-POMDP），每个店铺视为一个独立智能体，共享策略网络；训练使用全局数据，执行是分布式的。
- **状态**（`s_it`）：三个特征：待交付订单数（`d_it`）、骑手等待时间中位数（`cwt_it`）、局部供应压力（`sup_it`，经过店铺与区域中位数调整）。
- **动作**（`a_it`）：离散乘数集合 {0.8, 0.9, 1.0, 1.1, 1.2}，乘数越大，优化器越倾向速度（减少批处理），越小越倾向批处理效率。
- **奖励**：基于延迟的配送结果，按区域聚合。定义：
  - ASAP（顾客感知延迟）= 下单到送达时间
  - XCAT（骑手超额活动时间）= 总骑手活动时间减去直接行驶时间
  - 区域奖励 = 对区域内所有配送的加权和取负（权重 α 用于 ASAP，β 用于 XCAT），α=0.9（实验采用），β=1-α？实际公式（3）中 α 和 β 是固定权重，论文未明确β值，但取α=0.9，暗示β=0.1或类似平衡。
- **离线训练**：
  - 使用**Double DQN**减少最大化偏差，目标值由行为网络选择动作，目标网络评估。
  - 加入**保守Q学习（CQL）** 正则项，惩罚动作价值对，避免对日志数据中支持弱的动作产生高估。
  - 总损失：`L = L_DDQN + η * L_CQL`，其中η正则强度，训练曲线显示CQL使MSE不低于纯DQN，起到保守作用。
  - 网络结构：两层MLP（3→16→16→5），ReLU激活。

### 算法流程（文字说明）
1. 离线数据收集：两轮受控区域 rollout（第一轮纯随机探索，第二轮混合策略 50% exploit + 50% explore），得到约 346万条转换样本。
2. 训练共享 Q 网络：使用 mini-batch 32，Adam 优化器，学习率 1e-3，γ=0.99，梯度裁剪 10，每2个 epoch更新目标网络，共30 epochs。
3. 部署：每个店铺独立获取本地状态，前向计算 Q 值，选择 argmax 动作，将该乘数传给优化器。
4. 实时在线开关实验验证。

- **效率提升**：OWA-RL 显著降低 CAT（-1.26秒，p=0.019）和 CWT（-0.86秒，p=0.004），增加批处理比例（+0.495 pp，p<0.001），且全时段与晚餐时段一致。

- **成功验证**：从延迟市场反馈离线学习的策略可以安全部署至线上，在保持客户体验的同时优化骑手效率。

## 7. 优点

- **实际部署价值**：在真实的超大规模物流系统中完成从离线学习到在线实验的全链路，证明了 RL 在复杂经济系统中的可行性。
- **方法设计巧妙**：
   - 通过**低维控制接口**（仅一个乘数）对现有优化器进行“软”干预，保持所有生产约束和安全保障。
   - **离线训练**利用日志数据，规避在线探索的巨大风险。
   - **集中训练+分布执行**模式解决了多智能体协作与扩展问题。

- **实验工程严谨**：使用 CUPED 方差缩减、聚类 p 值、开关实验设计、guardrail 监控，符合工业标准。
