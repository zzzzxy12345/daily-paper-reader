## 摘要
语言模型智能体通过多轮工具使用在解决现实任务方面越来越有效。然而，训练可靠的工具使用智能体在实践中仍然具有挑战性。虽然强化学习提供了一种在策略范式，使智能体从其自身环境交互中改进，但其有效性在很大程度上取决于训练任务分布。当任务在训练前固定时，任务分布可能越来越与策略不断发展的能力不匹配，导致许多滚动被花费在无信息的任务上。我们提出SENTINEL，一种故障驱动的强化学习框架，将求解器的滚动失败转化为针对性的训练任务。SENTINEL遵循控制器-提议器-求解器循环：控制器分析失败的轨迹并总结重复出现的错误模式，提议器生成强调这些弱点的可执行任务，求解器则在针对性任务上训练。在Tau2-Bench Retail上使用Qwen3-4B-Thinking-2507，SENTINEL将Pass^1从66.4提高到74.9，并且在Pass^k指标上优于通用合成任务上的强化学习。这些结果表明，模型失败为改进使用工具的语言模型智能体提供了一个有效且可扩展的针对性训练信号来源。

## Abstract
Language model agents are increasingly effective in solving realistic tasks through multi-turn tool use. However, training reliable tool-using agents remains challenging in practice. While reinforcement learning provides an on-policy paradigm for improving agents from their own environment interactions, its effectiveness depends heavily on the training task distribution. When tasks are fixed before training, the task distribution can become increasingly mismatched with the policy's evolving capabilities, causing many rollouts to be spent on uninformative tasks. We propose SENTINEL, a failure-driven reinforcement learning framework that turns the Solver's rollout failures into targeted training tasks. SENTINEL follows a Controller--Proposer--Solver loop: the Controller analyzes failed trajectories and summarizes recurring error patterns, the Proposer generates executable tasks that stress these weaknesses, and the Solver is trained on the targeted tasks. On Tau2-Bench Retail with Qwen3-4B-Thinking-2507, SENTINEL improves Pass\^{}1 from 66.4 to 74.9 and outperforms RL on general synthetic tasks across Pass\^{}k metrics. These results demonstrate that model failures provide an effective and scalable source of targeted training signal for improving tool-using language model agents.

  - 强化学习（RL）是在策略的，但通常依赖预定义的静态任务集。随着策略能力演变，固定任务分布与策略实际弱点之间的不匹配日益严重，导致大量rollout浪费在无信息任务上。
- **核心问题**：能否构建一个自适应的RL训练任务分布，使其在训练过程中根据当前策略的弱点动态调整？
- **本文含义**：提出**SENTINEL**框架，利用模型自身的失败轨迹（即失败的rollout）来诊断弱点，并生成针对性的训练任务，从而将RL训练集中在模型真正需要改进的方向上，提升工具使用代理的可靠性。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
SENTINEL是一个**故障驱动的强化学习循环**：模型（Solver）在执行任务时产生的失败轨迹被用于分析错误模式，然后生成专门针对这些模式的新任务，再用于下一轮RL训练。这样，训练分布随着模型能力的提升而不断调整。

### 关键技术细节：Controller–Proposer–Solver循环

1. **Solver（求解器）**：
   - 是正在优化的策略模型 $\pi_\theta$。
   - 在工具环境 $E$ 中执行任务，产生轨迹 $\tau = (x, s_0, a_1, o_1, \dots, a_T, o_T, s_T)$。
   - 使用GRPO（Group Relative Policy Optimization）更新，奖励包括任务成功信号和若干形状奖励（如动作级检查、重复工具调用惩罚、长度惩罚等）。

2. **Controller（控制器）**：
   - 核心组件，负责**分析失败轨迹**并**生成生成指令**（directive）。
   - **步骤**：
     - 收集一批rollout轨迹，去重并分为失败集 $F_t$（奖励低于阈值 $\rho$）和成功集 $S_t$。
     - 调用LLM分析器识别**重复出现的错误模式**（如错误物品选择、缺少动作、策略违规、过早终止等）。
     - 结合历史训练记录 $H_t$，生成**自然语言指令** $d_t$，包含当前错误模式、已掌握技能等，传递给Proposer。

3. **Proposer（提议器）**：
   - 基于Controller的指令 $d_t$，生成执行性的、针对当前弱点的训练任务。
   - **任务生成方法**：
     - **轨迹接地任务构建**：首先在环境中构造一条可行的工具调用轨迹 $\tau^*$（包括用户资料、订单历史等），确保每个中间状态可达。
     - **失败感知任务重写**：将轨迹重写为自然语言用户请求 $x^*$，专门强调Controller发现的失败模式（例如，如果存在物品选择错误，则生成包含相似产品变种的请求）。
   - 这样生成的每个任务具有自一致性：用户请求来源于有效轨迹，且参考答案就是该轨迹。

### 算法流程
1. Solver在环境中执行当前任务缓冲区中的任务。
2. 每固定步数，Controller分析rollout，提取失败模式。
3. Proposer根据模式生成新任务，加入缓冲区。
4. Solver继续训练（RL），然后回到步骤1。
- 该循环持续进行，使得训练分布自适应地集中于模型当前的弱点。

## 6. 主要结论与发现

- SENTINEL在Tau2-Bench Retail上显著优于Base和General RL：Pass^1从66.4提升至74.9，Pass^2从51.6到60.5，Pass^3从43.2到51.2。
- 当模型已经经过SFT（起点更高）时，通用RL反而导致性能下降（74.3→68.1），而SENTINEL继续提升至78.1，说明**故障驱动任务生成在模型已有一定能力时仍然有效**，而通用任务会导致不稳定的退化。
- 失败模式分析表明，随着训练进行，错误类型从简单的执行缺失转向更细粒度的策略和交互错误，证明了训练分布的自适应调整。
- 结论：模型失败是有效的、可扩展的针对性训练信号来源。

## 7. 优点

- **创新性**：提出基于模型实际错误自动生成训练数据的闭环框架，将失败诊断与任务生成和RL优化直接连接。
- **自适应性**：训练任务分布随模型能力动态变化，避免固定任务集的弊端。
- **实践价值**：在仅4B参数的模型上取得了显著提升，证明方法有效且可推广；在SFT基础上仍能提升，克服了通用RL在强起点下的退化问题。
- **可解释性**：通过Controller明确输出错误模式，使训练过程可控可解释。
- **任务生成质量**：采用轨迹接地+失败感知重写，确保任务可执行且针对性强。
