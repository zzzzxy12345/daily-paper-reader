## 摘要
多轮工具使用智能体需要在多轮交互中进行推理、调用工具并适应观察结果。对此类智能体进行后训练颇具挑战性，因为强化学习虽然匹配仅提示推理设置，但常受限于稀疏奖励和弱信用分配；而基于专家轨迹的监督微调虽提供密集过程监督，却可能过度约束模型至固定轨迹。为解决这一问题，我们提出PACT，一种针对多轮工具使用智能体的特权轨迹协同训练框架。其核心思想是仅将专家轨迹作为训练时的优化信号，而非推演时的提示。PACT保持推演生成仅基于提示，随后通过两种互补信号利用专家轨迹指导优化：一种是以轨迹为条件的强化学习替代函数，在专家轨迹上下文下评估仅提示推演；另一种是组件感知的监督微调损失，以退火强度监督推理前缀和工具调用。为减少对仅训练时轨迹上下文的过度依赖，PACT进一步引入仅提示锚定。我们还提供了一种隐式轨迹视角，将两个基于轨迹的目标联系起来，并解释专家轨迹如何在推演生成中不被使用的情况下指导优化。在FTRL、BFCL和ToolHop上的实验表明，PACT相比基于监督微调和强化学习的强基线方法持续提升，突显了特权轨迹协同训练对于多轮工具使用学习的价值。

## Abstract
Multi-turn tool-use agents must reason, call tools, and adapt to observations across several interaction turns. Post-training such agents is challenging, as reinforcement learning often suffers from sparse rewards and weak credit assignment despite matching the prompt-only inference setting, while supervised fine-tuning on expert traces provides dense process supervision but can over-constrain the model to fixed trajectories. To tackle this, we propose PACT, a Privileged trAce Co-Training framework for multi-turn tool-use agents. The key idea is to use expert traces only as training-time optimization signals rather than rollout-time hints. PACT keeps rollout generation prompt-only, then uses expert traces to guide optimization through two complementary signals: a trace-conditioned RL surrogate that evaluates prompt-only rollouts under expert-trace context, and a component-aware SFT loss that supervises reasoning prefixes and tool-calls with annealed strength. To reduce over-reliance on the training-only trace context, PACT further introduces a prompt-only anchoring. We also provide a latent-trace view that connects the two trace-based objectives and explains how expert traces can guide optimization without being used during rollout generation. Experiments on FTRL, BFCL, and ToolHop show that PACT consistently improves over strong SFT- and RL-based baselines, highlighting the value of privileged trace co-training for multi-turn tool-use learning.

# PACT: Privileged Trace Co-Training for Multi-Turn Tool-Use Agents 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究问题**：多轮工具使用智能体（Multi-Turn Tool-Use Agents）需要跨多轮交互进行推理、调用工具、整合观察结果并最终给出答案。对这类智能体进行后训练（post-training）面临两难困境：
  - **强化学习（RL）**：匹配推理时的提示独占（prompt-only）设置，但多轮工具使用中的奖励通常稀疏、信用分配薄弱（难于定位哪些中间决策导致失败）。
  - **监督微调（SFT）**：利用专家轨迹可提供密集过程监督，但容易过约束模型模仿固定轨迹，忽视其他可行路径；且仅模仿无法反映中间决策对最终结果的影响。
- **整体含义**：如何在不改变推理时输入条件（即不使用专家轨迹作为提示）的前提下，利用专家轨迹引导模型训练，同时避免过约束和稀疏奖励问题。论文提出 **PACT 框架**，将专家轨迹仅作为训练时的优化信号（privileged signal），而推演生成（rollout）保持与推理一致的提示独占形式。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **提示独占推演生成**：训练时模型仅基于原始提示 `x` 采样完成的多轮工具使用轨迹 `τ`，不接触专家轨迹。
- **专家轨迹仅用于优化**：在优化阶段，利用专家轨迹 `h*` 提供两种互补信号：
  1. **轨迹条件 RL 替代**（Trace-Conditioned RL Surrogate）：在专家轨迹作为额外上下文条件下计算策略比率，用于评估提示独占推演。
  2. **组件感知 SFT 损失**（Component-Aware SFT Loss）：仅对推理前缀和完整工具调用 token 提供监督，并随训练逐步退火，减少过模仿。
- **提示独占锚定**（Prompt-Only Anchoring）：随机将一部分提示路由到标准 RL 损失（无轨迹条件），降低对训练时特权上下文的过度依赖。

### 2.2 关键技术细节

#### （1）提示独占推演
- 对于每个训练提示 `x_i`，旧策略 `π_old` 采样 K 个推演轨迹 `τ_{i,k} ∼ π_old(·|x_i)`。
- 计算奖励和优势（基于 MatchTIR 的硬信用分配），保持与推理一致。

#### （2）轨迹条件 RL 替代
- 将专家轨迹与提示拼接作为特权上下文：`ch_i = [x_i; h*_i]`。
- 对推演中的每个 token `τ_{i,k,n}`，计算在特权上下文下的策略比率：
  ```
  r^h_{i,k,n}(θ) = π_θ(τ_{i,k,n} | ch_i, τ_{i,k,<n}) / π_old(τ_{i,k,n} | ch_i, τ_{i,k,<n})
  ```
- 结合原有优势 `A_{i,k,n}`，使用 GRPO 式的裁剪损失：
  ```
  L^h_RL(i) = -E[min(r^h A, clip(r^h, 1-ε, 1+ε) A)]
  ```

#### （3）组件感知 SFT 损失
- 仅监督模型可控的组件：**推理前缀**（通过语义段落划分的退火前缀，如 L4→L3→...→L0）和 **完整工具调用 token**（工具名称和参数）。
- 工具观测（来自环境）不作为监督目标。
- 推理监督按语义段落退火（避免切断语义单位），工具调用监督按损失缩放退火（`λ_a(s) = λ_start - Δλ * min(s/S, 1)`）。
- 损失加权：`L_SFT(i;s) = -1/Z_i Σ m_{i,n}(s) · log π_θ(h*_{i,n}|x_i, h*_{i,<n})`，其中 `m_{i,n}(s)` 为活动 mask。

#### （4）整体目标与提示独占锚定
- 对于每个提示，以概率 `q` 采样 `z_i ~ Bernoulli(q)`：
  - 若 `z_i=0`，使用轨迹联合损失：`L_i = L^h_RL(i) + β_sft L_SFT(i;s)`
  - 若 `z_i=1`，使用标准提示独占 RL 损失：`L_i = L^x_RL(i)`
- `q=0.4` 为默认值，平衡轨迹引导与推理泛化。

#### （5）潜变量轨迹视角（Latent-Trace View）
- 将提示独占推演的对数似然分解为：`log π(τ|x) ≥ log p(h*|x) + log π(τ|x, h*)`
- 前者对应组件感知 SFT（学习轨迹结构），后者对应轨迹条件 RL（评估推演），解释了两部分互补性。

## 6. 主要结论与发现

1. **PACT 在所有三个基准上一致超越最强的 SFT 和 RL 基线**。例如，Qwen3-1.7B 平均分从 MatchTIR 的 21.70 提升至 24.90；Qwen3-4B 从 33.07 提升至 36.90。
2. **轨迹条件 RL 单独优于标准 RL**（Table 2），但直接加 SFT 会下降；两者组合最佳，证明互补性。
3. **特权轨迹组件全部重要**：移除工具响应导致最大下降（Solve-F1 从 33.21 降至 25.94），推理和工具调用也各有贡献。
4. **语义段落级退火优于 token 级退火**；工具调用缩放退火优于固定或 token 级退火。
5. **提示独占锚定（q=0.4）** 平衡了轨迹引导与推理泛化，完全不用或使用过多都会降低性能。
6. **潜变量轨迹视角** 为两个目标提供了统一理论解释。

## 7. 优点

- **保持推演与推理一致**：PACT 不使用专家轨迹扰动推演分布，确保训练与推理输入条件一致。
- **利用专家轨迹提供丰富过程监督**：轨迹条件 RL 提供全局上下文评估，组件感知 SFT 提供局部过程指导，退火机制避免过约束。
- **灵活的设计**：通过提示独占锚定、可调整的退火调度、组件 mask，三者协同可平衡信号。
- **理论视角清晰**：潜变量轨迹分解解释了为什么两部分损失互补。
- **消融实验全面**：覆盖了每个核心设计的选择，验证了有效性。

## 8. 不足与局限

1. **依赖专家轨迹数据**：需要教师模型（如 Qwen3-235B）生成并验证轨迹，且需人工/自动分割语义段落，增加了离线数据准备成本。
2. **模型规模有限**：仅测试 1.7B 和 4B 模型，在更大模型（如 7B/14B/更大）上的表现未知，结论可能无法直接推广。
3. **基准覆盖不足**：仅 FTRL、BFCL、ToolHop 三个基准，可能无法代表所有工具使用任务（如长尾 API、动态环境）。
4. **退火调度简单**：推理前缀使用固定语义等级顺序（L4→L0），工具调用缩放为线性衰减，未探索更自适应的调度（如基于学习进度动态调整）。
5. **缺少与其他深度混合方法的定量比较**：如与 BRIDGE、MIFO、CHORD 等更细致的消融（但已在主要基线中包含了 CHORD）。
6. **计算资源细节不完整**：未报告训练时长、使用的 GPU 小时数等，不利于复现和评估效率。
7. **潜在偏差风险**：教师轨迹可能包含偏见或错误模式（即使验证了答案），PACT 可能继承这些偏差；且语义分割依赖于 LLM 注释器，其质量影响训练。
