## 摘要
在非结构化环境中构建持久化的具身智能体，需要对跨越网络（API、物联网）和物理（操作、导航）领域的异构工具进行统一编排，同时具备从长时间运行中不可避免出现的物理故障中自主恢复的能力。现有系统将这些视为独立问题：基于VLM的规划器缺乏统一的网络-物理动作空间，智能体框架积累无界上下文导致时间连贯性下降，而VLA策略以开环方式执行，无法检测自身故障。我们认为，持久化自主性需要的不是单一模型，而是一种具有明确规划、记忆和验证分离的分层异步架构。为此，我们提出OmniAct框架，它集成了用于跨统一动作空间进行技能路由的多模态语义规划器、具有事件边界驱动压缩的自适应分层记忆（实现亚线性上下文增长），以及在物理执行期间闭合语义环的异步视觉抢占引擎。在两个机器人平台协调四个物联网设备的40个真实世界长时任务中，OmniAct在所有复杂度级别下均实现了端到端成功率的持续提升，在超过10万累计交互令牌下保持近乎平坦的令牌消耗，并将中等规模的开源模型提升至专有模型的性能水平。

## Abstract
Building persistent embodied agents in unstructured environments demands unified orchestration of heterogeneous tools spanning both cyber (APIs, IoT) and physical (manipulation, navigation) domains, coupled with autonomous recovery from physical failures that inevitably arise over extended operation. Existing systems treat these as separate problems: VLM-based planners lack a unified cyber-physical action space, agent frameworks accumulate unbounded context that degrades temporal coherence, and VLA policies execute open-loop without detecting their own failures. We argue that persistent autonomy requires not a monolithic model but a hierarchical asynchronous architecture with explicit separation of planning, memory, and verification. To this end, we present OmniAct, a framework integrating a multimodal semantic planner for skill routing across unified action spaces, an adaptive hierarchical memory with event-boundary-driven compression for sub-linear context growth, and an asynchronous visual preemption engine that closes the semantic loop during physical execution. Across 40 real-world long-horizon tasks on two robotic platforms coordinating four IoT devices, OmniAct achieves consistent improvements in end-to-end success across all complexity levels, maintains near-flat token consumption over under 100k+ accumulated interaction tokens, and elevates mid-scale open-weight models to proprietary-level performance.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有具身智能体在非结构化环境下的持久化部署面临三大根本性挑战：
    - **行动空间割裂**：网络工具（API、IoT）与物理操控（操作、导航）被当作独立系统处理，缺乏统一的网络-物理行动空间，导致自然语言指令中隐含的跨域协调需求无法被满足。
    - **记忆无界增长**：传统智能体框架线性累加交互历史，上下文迅速膨胀远超模型容量，长期偏好与近期状态产生语义冲突。
    - **开环执行与无法自主恢复**：VLA（视觉-语言-动作）策略以开环方式执行，无法检测自身物理失败（如抓取滑落、物体偏移），单次未检测的失败会级联导致整个任务不可逆。

- **整体含义**：实现真正的日常物理自主，不是通过单一端到端模型，而需要一种分层异步架构，将规划、记忆与验证明确解耦为专门模块，分别应对高层次的技能路由、长时状态跟踪与实时物理验证。

## 2. 方法论

- **核心思想**：提出 **OmniAct** 框架，通过三个紧密协作的模块实现“全模态动作”（omni-action），桥接全模态感知（语音、视觉、语言）与统一网络-物理行动空间。

- **关键技术细节**：
    1. **多模态语义规划器**（Multimodal Semantic Planner）：
        - 实例化高层规划策略，接收上下文 `ct` 和记忆 `Ht`，输出动作元组 `at = (τt, gt)`，其中 `τt` 选择目标技能，`gt` 指定技能参数。
        - 功能：解释用户意图、将长时指令分解为子目标、从历史记忆中避免已观测失败、支持并行多技能调用（输出结构化JSON规格）。
    2. **异步视觉抢占引擎**（Asynchronous Visual Preemption）：
        - 在物理执行期间，以低频率周期性采样视觉帧并提交给VLM进行语义级进度评估与安全检测。
        - 若检测到异常（如物体未被抓取、机器人未接近目标），立即中断VLA执行器，打包失败上下文返回规划器进行重规划。
        - 触发同时生成失败反思（因果分析与纠正策略）存入长期记忆。
    3. **自适应分层记忆**（Adaptive Hierarchical Memory）：
        - **感官记忆**：轻量级FIFO队列，缓存近期视觉帧，与规划上下文解耦。
        - **情景记忆**：通过事件边界驱动压缩，将原始观测序列压缩为紧凑关键帧摘要（保留语义、状态转移、动作结果），上下文随语义事件数亚线性增长。
        - **反思记忆**：存储静态先验（工具模式、技能清单）和动态经验（失败根因与纠正策略），在后续规划时检索注入。

- **算法流程（文字说明）**：
    - 系统接收多模态观测（视觉帧、离散状态、语音指令）。
    - 多模态语义规划器结合当前上下文和分层记忆输出下一个动作（技能+参数）。
    - 离散网络命令通过确定性返回码验证；连续物理命令转发给VLA策略执行。
    - 同时，异步视觉监控器周期性采样帧并调用VLM进行语义验证，若检测异常则中断执行并触发重规划。
    - 每次执行结果（成功/失败）通过事件边界检测更新情景记忆和反思记忆。

## 6. 主要结论与发现

1. **端到端成功率显著提升**：在所有复杂度级别（L1–L3），OmniAct 均大幅超越基线，尤其 L3 任务（操控 50.0%，导航 54.2%），而最佳基线（SayCan/Code as Policy）分别只有 43.8% 和 37.5%。
2. **子任务成功率更高**：L3 子任务成功率分别达到 80.5%（操控）和 80.6%（导航），说明异步视觉抢占和反思记忆将灾难性失败转化为可恢复延迟。
3. **token消耗近恒定**：在超过 100k 累积交互令牌的长时部署中，OmniAct 的每步输入令牌维持在 2.5k–10k 的窄带内，而原始历史线性增长至 120k+，避免了上下文溢出。
4. **跨模型通用性强**：OmniAct 提升了所有具备足够基线能力的骨干模型，尤其将中等规模开源模型（Qwen3-VL-30B-A3B）从 50.0% 提升至 80.0%，达到专有模型水平（Gemini-3.1-Pro 77.6%）。
5. **跨平台迁移成功**：在机械臂和移动机器人两种形态上取得一致改进，无需平台特定修改，证实框架抽象层次高于具体形态。

## 7. 优点（亮点）

- **统一行动空间与技能路由**：首次将网络工具（API、IoT）和物理控制纳入同一事件驱动循环，支持自然语言指令下的跨域协调。
- **异步视觉抢占引擎**：以语义级验证（而非底层几何约束）检测物理失败，填补了开环VLA策略的关键缺失，且通过异步、低频率设计平衡了计算成本。
- **事件边界压缩记忆**：利用物理交互的自然语义边界进行压缩，实现亚线性上下文增长，避免了固定窗口或均匀压缩的信息丢失。
- **双平台实物验证**：覆盖两种截然不同的机器人形态和多个IoT设备，验证框架的实际通用性。
- **开源模型提升显著**：无需额外训练，仅通过结构化编排即可提升中型开源模型至专有模型水平，具有实际部署意义。

## 8. 不足与局限

- **固化的低层策略**：高层规划器可通过反思记忆重规划，但无法改进底层VLA策略本身（如抓取精度限制导致的反复失败）。
- **视觉监控延迟窗口**：异步监控采样频率低于控制频率，在两次VLM推理之间存在残留错误动作的窗口，在涉及脆弱物体或人机共融场景中存在安全隐患。
- **骨干模型下限约束**：弱骨干（如Qwen3-VL-8B）输出畸形的结构化JSON，框架无法有效利用，限制了在极小模型上的适用性。
- **技能库静态**：系统不具备动态获取新技能的能力，无法应对超出预先定义技能组合的陌生任务。
- **实验环境限制**：所有任务在实验室受控场景中执行，未在真实家庭中与真实用户长期交互；任务集由作者设计，可能无意中包含有利于框架的细节；不同任务之间的难度均衡性未量化验证。
