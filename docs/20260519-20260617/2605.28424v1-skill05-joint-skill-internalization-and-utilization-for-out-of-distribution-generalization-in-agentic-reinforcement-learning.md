## 摘要
将显式技能赋予大语言模型已成为使自主智能体解决复杂任务的一种有前景的范式。智能体技能可自然分为用于广泛认知迁移的通用技能和用于动态执行的特定任务技能。然而，现有的基于技能的强化学习方法通常强制在完全外部化（导致高昂的上下文开销）和完全内化（存在过拟合和知识冲突风险）之间做出刻板选择。为了解决这一困境，我们提出Skill0.5，一种新颖的智能体强化学习框架，通过结合通用技能内化与特定任务技能利用来明确区分技能处理方式。在动态的、难度感知路由器的驱动下，Skill0.5将任务分流到不同的掌握层级以应用定制化优化策略：它通过特权蒸馏内化通用技能，为困难任务构建认知基础，同时利用对简单任务的诊断探针惩罚捷径并强制特定技能利用。在ALFWorld和WebShop上的实验表明，Skill0.5超越了基于记忆和基于技能的强化学习基线，在分布内和分布外场景中均实现了性能提升。

## Abstract
Equipping large language models with explicit skills has emerged as a promising paradigm for enabling autonomous agents to solve complex tasks. Agent skills can be inherently divided into general skills for broad cognitive transfer and task-specific skills for dynamic execution. However, existing skill-based reinforcement learning (RL) methods typically force a rigid choice between full externalization, which incurs prohibitive context overhead, and full internalization, which risks overfitting and knowledge conflicts. To address this dilemma, we propose Skill0.5, a novel agentic RL framework that explicitly differentiates skill treatments by combining general skill internalization with task-specific skill utilization. Driven by a dynamic, difficulty-aware router, Skill0.5 streams tasks into distinct mastery tiers to apply tailored optimization strategies: it internalizes general skills via privileged distillation to build a cognitive foundation for hard tasks, while using diagnostic probing on easy tasks to penalize shortcuts and enforce specific skill utilization. Experiments on ALFWorld and WebShop demonstrate that Skill0.5 outperforms both memory-based and skill-based RL baselines, yielding performance improvements across both in-distribution and out-of-distribution scenarios.

- **核心困境**：现实部署中技能库会动态增长，智能体需要同时处理已内化的通用认知和不断变化的任务特定规则，但现有方法对两类技能不加区分地统一处理，无法兼顾灵活性与稳定性。  
- **论文目标**：提出一种差异化的技能处理方案——**通用技能应内化**以节省上下文、构建稳定的认知基础；**特定任务技能应动态利用**（保持外部可插拔）以保证对新任务的适应性——从而提升智能体在分布外（OOD）泛化能力。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **Skill0.5** 是一个统一的智能体强化学习框架，基于**难度感知路由器**，根据智能体对任务的实时掌握程度（基于经验通过率）将任务动态分流到三个掌握层级，对每个层级应用定制化的优化目标：  
  - **困难任务（Hard）**：完全失败（通过率 = 0） → 内部化通用技能（特权蒸馏）。  
  - **中等任务（Medium）**：通过率高于0但低于动态阈值 → 标准GRPO强化能力。  
  - **简单任务（Easy）**：通过率高于动态阈值 → 反捷径探测，强制特定技能的真正利用。  

### 关键技术细节

1. **两阶段流程**：
   - **Phase-1（难度感知路由）**：对每个任务采样 G 个轨迹（使用仅含检索到的特定技能的“标准提示”），计算经验通过率 p_i。再基于滑动窗口平均阈值 η_t 将任务分为 Hard (p_i=0)、Medium (0<p_i≤η_t)、Easy (p_i>η_t)。
   - **Phase-2（层级定制优化）**：
     - **Hard 层**：使用包含通用技能的“特权提示”重新采样，筛选成功轨迹，通过**令牌级 Jensen-Shannon Divergence (JSD)** 蒸馏，使学生策略（仅用标准提示）模仿教师策略（用特权提示）的推理步骤，实现通用技能内化。
     - **Medium 层**：直接复用 Phase-1 的轨迹，使用标准的 GRPO（组相对策略优化）进行强化学习，目标为最大化任务成功率。
     - **Easy 层**：引入“无技能提示”（c_none = ∅）进行诊断性采样，得到无技能通过率 p_none。定义**利用增益** u_i = p_i - p_none，并计算任务级**利用优势** Au_i，与轨迹级优势叠加形成复合优势，通过 GRPO 优化。当 u_i 低于滑动平均时，负的 Au_i 会压制该任务的梯度，惩罚绕过特定技能的捷径行为。

2. **整体目标函数**：三个层级的损失（L_hard + L_medium + L_easy）联合优化，每个任务仅被一个层级的损失处理（互斥）。

3. **推理阶段**：智能体仅使用检索到的特定技能（通用技能已内化），从而同时避免上下文过长和参数冲突。
