## 摘要
本文提出了一种网络分布式多智能体强化学习（ND-MARL）框架，用于四旋翼无人机的一致性控制。与依赖集中式规划或完全去中心化执行的传统多智能体MARL方法相比，ND-MARL将群体通信图纳入决策过程。在2-邻居通信拓扑下，每个智能体仅观察两个邻居的信息，并通过分布式策略输出动作。使用多智能体软演员-评论家（MASAC）训练一个高级分布式一致性规划器，并将其嵌入层次化堆栈中，以生成由低级四旋翼控制器跟踪的参考目标位置。结果表明，与集中式MARL控制器相比，该方法实现了平滑的一致性轨迹和规划器-跟踪器集成。最值得注意的是，学习到的控制器展现出零样本可扩展性，即在三智能体系统上训练的策略无需重新训练或微调，即可部署到多达250个智能体的集群中，在相同的2-邻居通信拓扑下，由于稀疏信息传播，在大规模团队中实现了稳定的收敛，但稳态散布会增加。这些发现突显了ND-MARL作为分布式、通信感知的四旋翼一致性控制的稳定框架。

## Abstract
This paper proposes a Network Distributed Multi-Agent Reinforcement Learning (ND-MARL) framework for quadcopter consensus control. Compared to conventional multi-agent MARL formulations that rely on centralized planning or fully decentralized execution, ND-MARL incorporates the swarm communication graph into the decision process. Under a 2-Neighbor communication topology, each agent observes information of only two neighbors and outputs an action through a distributed policy. A high-level distributed consensus planner is trained using Multi-Agent Soft Actor-Critic (MASAC) and embedded in a hierarchical stack to generate reference target positions tracked by a low-level quadcopter controller. Results demonstrate smooth consensus trajectories and planner-tracker integration when compared to a centralized MARL controller. Most notably, the learned controller exhibits zero-shot scalability, as policies trained on a three-agent system are deployed to swarms of up to 250 agents under the same 2-Neighbor communication topology without retraining or fine-tuning, achieving consistent convergence with increasing steady-state spread at large team sizes due to sparse information propagation. These findings highlight ND-MARL as a stable framework for distributed, communication-aware quadcopter consensus control.

多无人机系统在灾害响应、监视、地图构建等任务中日益重要，但实际部署面临多个挑战：实时避碰、分布式决策以及共享目标的一致性达成。现有的多智能体强化学习（MARL）方法可分为三大类：

这些方法均未充分利用局部通信图的结构信息。论文的研究动机是：设计一种**网络分布式MARL框架（ND-MARL）**，将通信图显式纳入决策过程，使每个智能体仅与固定数量邻居交换信息，从而在保持分布式执行的同时实现全局一致性，并具备零样本可扩展性。

### 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：将全局一致性问题分解为局部邻域内的子问题。通过固定度（2-邻居）的无向通信图，每个智能体只观察两个邻居的相对状态，并输出本地的平面加速度，经过两次积分得到目标位置。高层规划器采用**多智能体软Actor-Critic（MASAC）**算法训练，嵌入层次化控制栈中，由低层单智能体RL推力矢量控制器跟踪。

**关键技术细节**：
- **状态空间**（每个智能体 \(i\) 的观测）：  
  \[
  S_i = [v_x, v_y, p_{ij}, p_{ik}, v_{ij}, v_{ik}], \quad (j,k) \in \mathcal{N}(i)
  \]
  其中 \(p_{ij}, v_{ij}\) 是智能体 \(i\) 相对于邻居 \(j\) 的位置和速度，所有量归一化到 \([-1,1]\)，实现平移不变性。
- **动作空间**：平面加速度 \([a_x, a_y] \in [-1,1]\)。
- **奖励函数**（局部 \(R_i\)）：  
  - 一致性奖励 \(r_i^{\text{cons}} = 1 - \tanh(\tilde{d}_i / \kappa_c)\)，\(\tilde{d}_i\) 为智能体到邻居的平均距离归一化值。  
  - 速度奖励 \(r_i^{\text{vel}}\)（仅在达成一致性后激活）：鼓励静止。  
  - 稀疏惩罚 \(r_i^{\text{pen}}\)：若已达成一致性后又失去，则惩罚 \(-1\)。  
  - 局部总奖励 \(R_i = w_c r_i^{\text{cons}} + w_v r_i^{\text{vel}} + r_i^{\text{pen}}\)，其中 \(w_c=1.0, w_v=0.5\)。全局奖励为各智能体局部奖励之和，满足式(1)的因子分解条件。
- **网络架构**：策略网络和值网络采用2层256节点MLP，ReLU激活。使用MASAC的熵正则化促进探索。
- **层次化控制**：高层MARL规划器输出目标位置；低层使用单智能体推力矢量控制器（文献[16]）跟踪；再经姿态PID生成电机指令。低层状态包含姿态、线速度、角速度、位置误差等12维。

**关键优势**：由于观测和动作复杂度均为 \(O(1)\)，计算和通信成本不随智能体数量增长。
