## 摘要
强化学习在部署到与训练环境不同的环境中时，通常会面临性能下降的问题。现有技术如域随机化（DR）可以缓解这一问题，但需要访问多样化的训练环境和完整的轨迹可观测性，这些假设在隐私保护或受限场景中不成立，因为在这些场景中只能获得标量性能指标。我们提出通过进化奖励塑造进行泛化（GERS），这是一种双层优化方法，仅使用验证环境的标量反馈来提高在未见测试环境上的泛化能力。在下层，由上层塑造的奖励函数引导的强化学习智能体在有限的可访问轨迹数据的训练环境上学习策略；在上层，CMA-ES优化奖励塑造参数，以最大化无法获取轨迹数据的独立验证环境上的累积未塑造奖励。连续控制任务的结果表明，GERS在未见测试环境上优于标准强化学习基线。尽管DR将GERS的训练和验证环境集合视为一个需要轨迹访问的单一训练集，而GERS无法访问验证轨迹，但GERS的性能与DR相当。这些结果证实，GERS在受限数据访问约束下有效地增强了泛化能力。

## Abstract
Reinforcement learning (RL) often suffers from performance degradation when deployed in environments that differ from those encountered during training. Existing techniques such as domain randomization (DR) mitigate this, but require access to diverse training environments and full trajectory observability, assumptions that fail in privacy-preserving or restricted scenarios where only scalar performance metrics are available. We propose Generalization via Evolutionary Reward Shaping (GERS), a bilevel optimization approach to improve generalization on unseen test environments using only scalar feedback from validation environments. At the lower level, an RL agent guided via a reward function shaped by the upper level learns a policy on a limited set of training environments with accessible trajectory data; at the upper level, CMA-ES optimizes the reward shaping parameters to maximize the cumulative unshaped reward on separate validation environments for which trajectory access is unavailable. Results on continuous control tasks indicate that GERS outperforms the standard RL baseline on unseen test environments. GERS performance is comparable to DR, despite DR treating the combined set of training and validation environments of GERS as a single training set that requires trajectory access, whereas GERS cannot access validation trajectories. These results confirm that GERS effectively enhances generalization under restricted data access constraints.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：强化学习（RL）在部署时，如果测试环境的动力学（如质量、摩擦、重力等物理参数）与训练环境存在差异，会出现严重的性能下降（泛化差距）。
- **已有方法的局限**：域随机化（DR）等方法虽然能提升泛化，但要求训练期间能访问大量多样环境的完整轨迹（状态-动作-奖励序列）。这在隐私保护场景（如自动红队演练中客户的生产副本）或分布式数据受限场景下不成立——开发者只能获得验证环境的标量回报（如累积未折扣奖励），无法观察轨迹。
- **整体含义**：探索仅利用验证环境的标量反馈来塑造训练环境的奖励函数，从而提升策略在未见测试环境上的泛化能力，为数据访问受限的现实应用提供可行方案。

## 2. 方法论
### 2.1 核心思想
- 将泛化问题建模为**双层优化**（bilevel optimization）：
  - **下层**：在有限个训练环境（轨迹可观测）上，使用**经塑造的奖励函数**训练策略（PPO）。
  - **上层**：优化奖励函数的塑造参数，使得下层得到的最优策略在验证环境（轨迹不可观测，仅标量回报）上获得最高的累积回报。
- 通过进化策略（CMA-ES）仅利用验证环境的标量评分来更新上层参数，避免了对验证轨迹梯度的依赖。

### 2.2 关键技术细节
- **奖励塑造**：线性参数化形式：
  \[
  \hat{r}_{\nu}(s, a) = r(s, a) + \langle \nu, \phi(s, a) \rangle
  \]
  其中 \(\phi(s,a) = [s^\top, a^\top]^\top\) 为状态和动作的拼接特征，\(\nu\) 为待优化参数。
- **下层优化**：使用PPO在固定训练环境集上最大化带塑造奖励的折扣回报。
- **上层优化**：使用CMA-ES（dd-CMA-ES变体）优化\(\nu\)，最大化验证环境上的平均累积未折扣回报。
- **加速策略**：在每代CMA-ES开始时，使用上一代中验证得分最高的策略参数来初始化所有并行PPO运行，减少下层训练成本。
- **算法流程**（Algorithm 1）：
  1. 在训练环境上用原始奖励预训练策略\(K_{init}\)步，得到初始参数\(\theta^{(1)}\)。
  2. 对每代\(g=1,...,G\)：
     - 从CMA-ES采样\(\lambda\)个候选\(\nu_i\)。
     - 并行初始化PPO（演员-评论家）为\(\theta^{(g)}\)，用\(\hat{r}_{\nu_i}\)训练\(K\)步。
     - 在验证集上评估每个训练后策略的回报，作为适应度\(f(\nu_i)\)。
     - 用所有候选对\((\nu_i, f(\nu_i))\)更新CMA-ES。
     - 选择验证得分最高的策略作为下一代的初始\(\theta^{(g+1)}\)。
  3. 输出最终策略和最优奖励参数。

## 7. 优点

3. **自动化奖励塑造**：免去手工设计奖励函数的繁琐，自动优化以提升泛化。
4. **实验设计较严谨**：严格分离训练/验证/测试；每代重采样种子确保评分可比性；热力图全面展示性能边界。

1. **统计显著性不足**：仅5次试验，经多重比较校正后未达显著水平，结论稳健性有待加强。
